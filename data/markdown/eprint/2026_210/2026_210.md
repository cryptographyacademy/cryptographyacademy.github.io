# How to Classically Verify a Quantum Cat without Killing it

Yael Tauman Kalai<sup>∗</sup> Dakshita Khurana† Justin Raizes‡

#### Abstract

Existing protocols for classical verification of quantum computation (CVQC) consume the prover's witness state, requiring a new witness state for each invocation. Because QMA witnesses are not generally clonable, destroying the input witness means that amplifying soundness and completeness via repetition requires many copies of the witness. Building CVQC with low soundness error that uses only one copy of the witness has remained an open problem so far.

We resolve this problem by constructing a CVQC that uses a single copy of the QMA witness, has negligible completeness and soundness errors, and does not destroy its witness. The soundness of our CVQC is based on the post-quantum Learning With Errors (LWE) assumption.

To obtain this result, we define and construct two primitives (under the postquantum LWE assumption) for non-destructively handling superpositions of classical data, which we believe are of independent interest:

- 1. A state preserving classical argument for NP.
- 2. Dual-mode trapdoor functions with state recovery.

# <span id="page-2-0"></span>1 Introduction

Imagine a quantum prover that would like to demonstrate possession of Schr¨odinger's proverbial cat without killing the cat. Or picture a classical verifier seeking to assess whether a prover has sufficient quantum money to pay them. In both cases, the challenge is: how can one prove the existence of a valuable quantum state without destroying it in the process?

It is well-known that quantum states are precious, consumable resources. Some—like magic states—can be produced at a nontrivial but manageable cost, while others may be far harder to obtain. In particular, it is computationally infeasible to duplicate QMA witnesses, relative to an oracle [\[CKP25\]](#page-63-0). Since only one copy of a witness may be available, it is crucial to avoid destroying this witness while proving statements about it.

Non-Destructive Classical Verification of QMA. Mahadev's breakthrough [\[Mah18\]](#page-64-0) opened the door to classical verification of quantum computation (CVQC), enabling a quantum prover to prove QMA statements to an efficient classical verifier. This work spurred a number of exciting developments in CVQC [\[Mah18,](#page-64-0) [ACGH20,](#page-61-1) [CCY20,](#page-62-0) [BKL](#page-62-1)<sup>+</sup>22, [Zha22,](#page-64-1) [KLVY23,](#page-63-1) [MNZ24,](#page-64-2) [GKNV25,](#page-63-2) [BKM](#page-62-2)<sup>+</sup>25, [BK25\]](#page-62-3). And yet, despite this progress, a fundamental limitation remains: all existing protocols for classically verifying QMA statements end up irreversibly destroying the prover's precious witness in the process. In this work, we ask

> Can a quantum prover convince a classical verifier of a QMA statement without sacrificing the witness?

Using a Single Witness. In the setting where the verifier is quantum, non-destructive verification is exactly what enables deciding QMA with negligible error using only one copy of a witness. If a verifier can re-test the same witness repeatedly, completeness and soundness errors can be driven to 2<sup>−</sup><sup>λ</sup> without increasing the witness length. This idea is at the core of Marriott-Watrous's fundamental result amplifying QMA with one copy of the witness [\[MW05\]](#page-64-3). Conversely, this near-perfect completeness also enables non-destructive QMA verification with a quantum verifier. Since the witness is accepted in [\[MW05\]](#page-64-3) with overwhelming probability, the acceptance measurement disturbs it only negligibly.

In contrast, existing protocols for classically verifying QMA typically need sequential or parallel repetition to reduce completeness/soundness errors, but each repetition irreversibly damages the witness. As a result, these protocols require the prover to start with many copies of the QMA witness, a demand which is often unrealistic. We ask:

> Can a quantum prover convince a classical verifier of a QMA statement using only one copy of the QMA witness?

This Work. Our work answers both questions in the affirmative for CVQC, aligning the case of classical verification with the QMA picture. We note that unlike the QMA setting, even if near-perfect completeness were achieved in CVQC protocols, it is unclear whether the prover's witness would be preserved in general.

{3}------------------------------------------------

In fact, our main result offers the best of both worlds: a CVQC for QMA with negligible error that uses a single copy of a Marriott-Watrous witness [\[MW05\]](#page-64-3), and where the prover ends the protocol with a witness that has only negligible statistical distance from its original witness.

We achieve our main result via two generic transformations, both of which rely on the post-quantum hardness of LWE:

- 1. Witness-Preservation for Near-Perfect Completeness: We compile any nonadaptive[1](#page-3-1) CVQC with 1−negl(λ) completeness into a witness-preserving (non-adaptive) CVQC, where an honest prover ends with a state negligibly close to their original witness.
  - This transformation reduces our task to constructing a non-adaptive CVQC that has near-perfect completeness and uses a single witness (but may destroy the witness).
- 2. Completeness and Soundness Amplification via Non-Destructive Verification: We compile any non-adaptive CVQC for a language in QMA1−2−λ,2−<sup>λ</sup> (where the honest prover uses a single witness that is accepted by the QMA verifier with overwhelming probability) into one with 1 − negl(λ) completeness and negl(λ) soundness. This compiler first converts any non-adaptive CVQC into one that only mildly destroys the prover's witness, then it amplifies soundness and completeness by sequential repetition, using the resulting witness which is still "good enough."

We obtain our main result by taking any non-adaptive CVQC that uses a single copy of the witness, and first applying the second compiler that amplifies soundness and completeness (while still using a single witness), and then applying the first compiler that makes the CVQC witness preserving.

### <span id="page-3-0"></span>1.1 Results.

Our main result is a CVQC which uses one copy of the witness and preserves it.

Theorem 1 (Informal). There exists a CVQC protocol for any language L ∈ QMA1−2−λ,2−<sup>λ</sup> with the following properties:

- One-Copy Amplified Soundness and Completeness. It has 1 − negl(λ) statistical completeness and negl(λ) computational soundness error, assuming the postquantum hardness of LWE, using one copy of any witness |w⟩ for L ∈ QMA1−2−λ,2−<sup>λ</sup> . Furthermore, it is an argument of knowledge.
- Witness Preserving. At the end of the protocol, the prover is left with a state that is statistically close to |w⟩.

<span id="page-3-1"></span><sup>1</sup>A non-adaptive CVQC is one where the verifier's messages are independent of the prover's. This is a broad class containing all CVQC protocols we are aware of and which naturally generalizes public-coin protocols.

{4}------------------------------------------------

Remark 1. The prover may use one copy of any witness |w⟩ which is accepted with probability 1−2 −λ . Thus, if short, high-quality witnesses exist, the CVQC prover may use a short witness. In contrast, prior works require multiple copies of |w⟩ regardless of how high-quality it is.

Marriot and Watrous [\[MW05\]](#page-64-3) show that every QMAa,b language, where a−b = 1/poly(|x|), has short witnesses which are accepted with probability 1 − 2 −λ . It should be noted that Marriot-Watrous witnesses have a special form and not every QMAa,b witness is a QMA1−2−λ,2−<sup>λ</sup> witness. This limitation is inherent since thresholds below 1 − negl(λ) allow the possibility of a non-negligible amplitude on a "junk" state which cannot be amplified.

Being simultaneously an argument of knowledge and witness preserving may at first seem contradictory; if it is possible to extract the witness from the prover and have the prover keep their witness, have we not cloned the witness, which could in general be hard for QMA [\[CKP25\]](#page-63-0)? Indeed, [\[VZ21\]](#page-64-4) formalizes such an intuition for non-destructive proofs of knowledge (i.e., with statistical soundness).

Our approach dodges this issue by ensuring that extraction and preservation are both possible at the start of the protocol, but do not happen simultaneously. In a real execution, the prover's witness is preserved. However, when extracting an adversarial prover's witness, the extractor interacts differently with the prover in a mode which disables witness preservation. An adversarial prover cannot distinguish which mode is occurring, making our CVQC both witness preserving and an argument of knowledge. See Section [2.1](#page-9-0) for more details.

One-Copy Amplification via Non-Destructive Verification. To obtain our main result, we build two compilers equipping any non-adaptive CVQC with the two desired properties. First, we show how to generically convert any non-adaptive CVQC with arbitrarily low completeness into one with near-perfect completeness and soundness, without using extra copies of the witness.

<span id="page-4-0"></span>Theorem 2 (Informal). Assuming the post-quantum hardness of LWE, there is an efficient compiler that converts any non-adaptive CVQC protocol with completeness c and soundness error s satisfying c − s ≥ 1/poly(λ) into a new CVQC protocol which has

• One-Copy Amplified Soundness and Completeness. It has 1 − negl(λ) statistical completeness and negl(λ) computational soundness error using one copy of the witness |w⟩ for the original CVQC.

Furthermore, if the original CVQC was an argument of knowledge, so is the new one.

We prove this theorem by first converting the original CVQC into one which partially preserves the witness. Specifically, at the end of the protocol the prover has a new witness that is at most ϵ worse than its original witness, where ϵ = 1/poly(λ) is a tunable parameter. Then we sequentially amplify completeness and soundness using the leftover witness.

To use the amplification compiler (Theorem [2\)](#page-4-0), we need to start with a non-adaptive CVQC that uses a single witness but may have poor completeness and soundness. The first

Witness-Preservation via Near-Perfect Completeness. Second, we show how to compile any non-adaptive CVQC with near-perfect completeness into a witness preserving CVQC, again without using extra copies of the witness. We obtain our main result by applying this second compiler to our first one that amplifies completeness and soundness.

<span id="page-5-1"></span>**Theorem 3** (Informal). Assuming the post-quantum hardness of LWE, there is an efficient compiler that converts any non-adaptive CVQC protocol with completeness  $1 - \mathsf{negl}(\lambda)$  and soundness error s into a new CVQC protocol with the same completeness and soundness which is

• Witness Preserving. At the end of the protocol, the prover is left with a state that is statistically close to its input witness  $|w\rangle$ .

Furthermore, if the original CVQC was an argument of knowledge, so is the new one.

To construct our compilers, we develop two techniques that we believe are of independent interest.

Witness-preserving arguments for NP. We construct an interactive argument for NP with the guarantee that if the prover starts with a superposition over possible witnesses  $\sum \alpha_w |w\rangle$  then this superposition is maintained at the end of the interactive argument. We refer to such an argument system as witness preserving.

**Theorem 4** (Informal). There exists a witness preserving interactive argument for NP assuming the post-quantum LWE assumption.

We refer the reader to Section 6 for details. The construction uses the following primitive, which is also used by our compilers.

Dual-mode trapdoor function family with state recovery. We define and construct a family of randomized functions that can be sampled in two modes: recovery mode or injective mode. It has the guarantee that if a function is sampled in the recovery mode then it is non-collapsing in the sense that applying f to a state  $\sum_{w} \alpha_{w} |w\rangle$  and measuring the output y, allows one to reconstruct the state  $\sum_{w} \alpha_{w} |w\rangle$  given a trapdoor. On the other hand, if a function is sampled in the injective mode then it is injective and hence the state is collapsed to a single w, which can be computed given the trapdoor.

Witness Preservation Against Malicious Verifiers. Finally, we mention the implications of our results to a related interesting scenario where the verifier is malicious and attempts to destroy the prover's witness. In other words,

Can a quantum prover convince a verifier of a QMA statement without losing their witness even if the verifier deviates from the protocol?

It is not hard to see that any witness-preserving CVQC (not necessarily non-adaptive) can be generically converted into one with malicious-verifier witness preservation, assuming time-lock puzzles and zero-knowledge arguments for NP. The verifier sends the prover a time-lock puzzle, containing its randomness; then for each message it sends in the CVQC, it proves that this message is consistent with the time-locked randomness in zero-knowledge.[3](#page-6-1) If the verifier ever deviates from the protocol (e.g. by aborting), the prover can solve the time-lock puzzle and complete the protocol locally to recover its witness. On the other hand, the security of the time-lock puzzle and zero-knowledge argument ensure that a real prover has no information about the verifier's randomness before the time-lock puzzle can be solved, so soundness is still guaranteed.

A conceptually even simpler approach uses classical witness encryption for QMA (for which there are no plain-model constructions, but there are constructions in the classical oracle model e.g. due to [\[BM22\]](#page-62-4)). The verifier encrypts a random message m under the QMA statement to be proven, then the prover decrypts it and sends back m′ , whereupon the verifier checks that m′ = m. If the witness encryption is correctly generated, then decryption is almost deterministic and disturbs the prover's witness negligibly. Honest encryption can be guaranteed using a zero-knowledge argument.

### <span id="page-6-0"></span>1.2 Related Works

Classical Proofs of Quantum Knowledge [\[VZ21\]](#page-64-4). Vidick and Zhang observed that proofs of quantum knowledge with classical communication are generally destructive unless the witness can be (inefficiently) cloned, and so an honest prover in a sequentially repeated protocol needs to use multiple copies of the witness in order to succeed. Their analysis applies to perfectly non-destructive, statistically sound proofs and they provide some discussion about extending the results to protocols which only damage the witness by a small amount.[4](#page-6-2) On the other hand, in this work we build arguments of knowledge that are non-destructive.

<span id="page-6-1"></span><sup>3</sup>For soundness, the zero-knowledge argument and CVQC must be together faster than the time to solve the time-lock puzzle.

<span id="page-6-2"></span><sup>4</sup>They mention that their no-go theorem does not apply to QMA since it is always possible to inefficiently find some witness. However, if the witness is not unique, it is still impossible to clone the exact witness used.

{7}------------------------------------------------

Proofs of No Intrusion [\[GR25\]](#page-63-3). Recently, Goyal and Raizes investigated a new primitive they called "proofs of no-intrusion" (PoNI). A PoNI for encryption is essentially a nondestructive classical argument that a ciphertext has not been "stolen", in the sense that no external party can decrypt the ciphertext even given the key. Our results enable these by combining our state-preserving argument for NP with a public-key encryption scheme with publicly verifiable certified deletion (known from just public-key encryption [\[BKM](#page-62-5)<sup>+</sup>23, [KNY23\]](#page-63-4)). See Appendix [A](#page-65-0) for more details.

However, their main new technical tool achieves something incomparable to our results. Goyal and Raizes introduce PoNIs for coset states, which is essentially a non-destructive classical argument of quantum knowledge specifically for coset states [\[CLLZ21\]](#page-63-5). Although conceptually related, PoNIs for coset states are incomparable to our results. Our results allow non-destructive arguments for the general class of QMA, where the statement being proven is public. On the other hand, PoNIs for coset states allow the prover to blindly prove a specific statement about coset states, where the prover does not know the statement being proven.

Disambiguation. [\[LMS22b\]](#page-64-5) also used the term "state preserving argument" in the context of arguments of knowledge. There, "state preserving" refers to the ability to extract some knowledge from an adversarial prover without noticeably disturbing the prover's internal state. This takes place in a sandbox where the communications with the adversarial prover can be purified or rewound. In contrast, our usage of state preserving refers to preserving an honest prover's input even in a real execution.

### <span id="page-7-0"></span>1.3 Open Problems

We conclude this section with a set of open directions:

- 1. Round Complexity. Can we achieve single-copy CVQC for QMA with negligible errors in constant (or even two) rounds of interaction? Since high-error protocols can only be composed sequentially when there is a single copy of the witness, addressing this question likely requires additional techniques beyond those developed in this work.
- 2. Succinctness. Can we achieve single-copy CVQC for QMA with negligible errors with smaller communication than the instance size? Our compiler from 1 − negl(λ) completeness to witness preservation can make each message succinct using techniques from [\[GKNV25\]](#page-63-2). However, our compiler for amplifying completeness using one copy of the witness uses a number of sequential repetitions which grows with the instance size, preventing it from being succinct even with short messages.

# <span id="page-8-0"></span>2 Technical Overview

In what follows, we describe our compiler that compiles any non-adaptive CVQC protocol, that uses a single copy of the QMA witness (but may have poor completeness and soundness parameters and may destroy the witness), into a CVQC protocol that uses a single copy of the QMA witness and is witness-preserving and has nearly perfect completeness and soundness. Our compiler works in three stages.

• Stage 1: ϵ-Non-Destruction and a Special Case. First, we compile the CVQC protocol to an ϵ-non-destructive protocol with almost the same completeness c−negl(λ) and soundness error s + negl(λ). Roughly speaking, if the prover starts with a witness |w⟩ which would be accepted by the QMA verifier with probability p (a "p-good witness"), then at the end of the protocol they have a witness |w ′ ⟩ which is at least (p − ϵ)-good.[5](#page-8-1) The runtime scales with 1/ϵ, so we are limited to ϵ = 1/poly(λ).

As a special case of this compiler, if the underlying CVQC starts with 1 − negl(λ) completeness, then we achieve witness preservation: at the end of the protocol |w ′ ⟩ is negligibly far from |w⟩. The runtime only blows up by a factor of the security parameter λ. This special case is used in Stage 3, below.

• Stage 2: Sequential Amplification. Next, we amplify completeness and soundness by repeating the protocol sequentially using the same witness. Since we can tune ϵ to any 1/poly(λ), if we start with a witness just slightly better than the QMA threshold, we can ensure that every intermediate witness still surpasses the QMA threshold. Thus, the CVQC verifier accepts with probability c − negl(λ) in each execution.

<span id="page-8-1"></span><sup>5</sup>Our ϵ-non-destructive protocol actually uses a special set of "repairable" witnesses that include all Marriot-Watrous witnesses and more generally any (1 − negl(λ))-good witness, but for now we make the simplifying assumption that the ϵ-non-destructive guarantee applies to any witness.

{9}------------------------------------------------

The parameters can also be tuned more aggressively to end up with a (p − ϵ ′ )-good witness |w ′ ⟩ after all executions, but we are still limited to ϵ ′ = 1/poly(λ) by the runtime blowup from stage 1.

• Stage 3: Witness Preservation. Finally, we re-compile the amplified protocol again, using the special case of the stage 1 compiler. Since the amplified protocol has 1 − negl(λ) completeness, the final compiled protocol is witness preserving.

The bulk of the technical ideas appear in stage 1, which is the focus of this overview.

### <span id="page-9-0"></span>2.1 Special Case: Almost-Perfect Completeness

We begin with the special case of c = 1−negl(λ). The main body of the compiler is the same as the general completeness version, but the witness recovery step is significantly simpler and illustrates the core ideas well.

Classical Verification in Superposition. Ultimately, the goal in CVQC is to convert a quantum statement into a classical one so that the classical verifier can check it. The problem with existing approaches to CVQC is that measuring said transcript collapses the prover's state. If the prover could keep the transcript "in superposition" throughout the protocol, then at the end of the protocol (after the verifier measures the verdict), the prover could uncompute the transcript and recover the witness. This yields something like the following very high-level approach.

1. Interactively, the prover coherently computes its answers a<sup>i</sup> to each of the verifier's ℓ queries q<sup>i</sup> during the CVQC and does not measure them. At the end, the prover holds a superposition over answers a[ℓ] = (a1, . . . , aℓ):

$$\sum_{a_{[\ell]}} \alpha_{a_{[\ell]}} |a_{[\ell]}\rangle$$

Since the verifier's queries are non-adaptive, it doesn't need to see the prover's answers to send the next query.

- 2. The verifier sends its (classical) secret state from the CVQC to the prover[6](#page-9-1) and the prover uses a special argument for NP to prove that it holds an answer sequence a[ℓ] which the verifier would accept, without collapsing its superposition over a[ℓ] .
  - Since the completeness is almost perfect, measuring whether a[ℓ] is an accepting transcript disturbs the state negligibly.
- 3. Finally, the prover uncomputes its answers P a[ℓ] α<sup>a</sup>[ℓ] |a[ℓ]⟩ to recover the witness.

<span id="page-9-1"></span><sup>6</sup>Sending the verifier's internal CVQC state is necessary so that the prover knows the NP statement it is proving. If the CVQC is publicly verifiable, this is not necessary, but publicly verifiable CVQC remains an open problem.

{10}------------------------------------------------

Of course, this approach is not sound as written because the prover can change its answers a[ℓ] after seeing future queries or even the verifier's internal state. So, the question is:

How do we bind the prover to a[ℓ] without collapsing it or otherwise preventing the prover from uncomputing its answers and recovering the witness?

Dual-Mode Trapdoor Functions with State Recovery. Our solution to this conundrum is an observation of a novel property of [\[BCM](#page-61-2)<sup>+</sup>18, [Mah18\]](#page-64-0)'s trapdoor claw-free functions which we abstract out as a new primitive: dual-mode trapdoor functions with state recovery. This is a family of randomized functions and trapdoors (fpp,tdpp). The tuples can be generated in two indistinguishable modes: recovery or injective.

In injective mode, two different inputs x ̸= x ′ cannot collide: fpp(x; r) ̸= fpp(x ′ ; r ′ ) for all randomness r and r ′ . Given any y = fpp(x; r), the trapdoor tdpp can be used to find x. This mode will be important for proving soundness of our protocol.

In recovery mode, fpp is 2<sup>n</sup> -to-1, where n is the input size, and tdpp can be used to invert a preimage (x; rx) of y = fpp(x; rx) for any x ∈ {0, 1} n , where r<sup>x</sup> is the randomness. Similarly to Mahadev's measurement protocol, a quantum party can "commit" to a computational basis measurement of a quantum state P <sup>x</sup> α<sup>x</sup> |x⟩ by preparing a superposition over all randomness r then coherently evaluating fpp(x; r) and measuring the result y. This results in the state

$$\sum_{x} \alpha_x |x, r_x\rangle$$

where fpp(x, rx) = y for all x ∈ {0, 1} n . Here is where the recovery comes in: if the trapdoor tdpp were revealed, the quantum party can use it to coherently compute r<sup>x</sup> controlled on x. By subtracting that from the randomness register, they disentangle their state and recover P <sup>x</sup> α<sup>x</sup> |x⟩.

Using the recovery mode, we can update the protocol above as follows.

1. The verifier generates a pair (fpp,tdpp) in the recovery mode, and sends pp to the prover. Interactively, the prover "commits" to each answer a<sup>i</sup> by evaluating fpp as discussed previously. At the end, the verifier knows y[ℓ] = (y1, . . . , yℓ) and the prover holds

$$\sum_{a_{[\ell]}} \alpha_{a_{[\ell]}} |a_{[\ell]}\rangle \otimes |r_{a_{[\ell]}}\rangle$$

where y<sup>i</sup> = fpp(a<sup>i</sup> ; r<sup>a</sup><sup>i</sup> ) for each i ∈ [ℓ].

- 2. The verifier reveals its secret state for the underlying CVQC and the prover uses the "state-preserving" argument for NP (which we elaborate on below) to prove that there exist preimages (a<sup>i</sup> ; r<sup>a</sup><sup>i</sup> ) of y[ℓ] such that the verifier would accept a[ℓ] in the CVQC.
- 3. The verifier sends tdpp to the prover, who uses it to uncompute each |r<sup>a</sup>[ℓ] ⟩. Since the superposition over answers a[ℓ] is no longer entangled with any external state, the prover can uncompute its answers to recover the witness.

If the state-preserving argument for NP does not disturb the superposition P a[ℓ] α<sup>a</sup>[ℓ] |a[ℓ]⟩ ⊗ |r<sup>a</sup>[ℓ] ⟩, then the recovery mode trapdoor allows recovering the prover's original witness up to negligible disturbance.

State-Preserving Arguments for NP. Roughly, we need the prover to be able to use a superposition of NP witnesses P <sup>i</sup> α<sup>i</sup> |wNP,i⟩ to prove the NP statement, and at the end of the argument they should have a state close to their original witness superposition.

A very simple construction is possible using witness encryption for NP. The verifier encrypts a random message m under the statement to be proven. Then, the prover coherently decrypts m′ using their witness superposition. The verifier accepts if m = m′ . Since decryption is almost deterministic using a valid witness, measuring m′ disturbs the prover's superposition negligibly. Unfortunately, witness encryption is a relatively strong assumption, which we would like to avoid (and do avoid).

The next idea is to use a statistically witness-indistinguishable (WI) argument. Intuitively, since the verifier's view is statistically independent of which witness was used, the prover avoids measuring their state. However, some care is needed here. During the argument, the prover may also entangle their witness with the randomness used. To recover the witness superposition, it is crucial that the prover be able to unentangle their state later.[7](#page-11-0)

We show that the well-known 3-coloring WI argument, when instantiated using dualmode trapdoor functions with state recovery, is a state-preserving argument for NP. The analysis requires a careful accounting of the prover's randomness – both for the trapdoor function and for randomness inherent to the 3-coloring protocol – to show that it can be safely uncomputed at the end of the protocol.

Soundness. The proof of soundness reduces to the soundness of the underlying CVQC by extracting an accepting transcript a[ℓ] from the prover. As a result, the new CVQC is a proof of knowledge if the underlying CVQC is. If (fpp,tdpp) is generated in injective mode, then there is only one preimage a<sup>i</sup> of each y<sup>i</sup> in step 1. The reduction can use tdpp to extract each answer a<sup>i</sup> from y<sup>i</sup> as the prover sends it, then forward a<sup>i</sup> to the CVQC verifier.

The subtle part of the proof is in showing that the extracted a[ℓ] will be accepted by the verifier. Intuitively, the indistinguishability of injective and recovery mode ensures that the adversarial prover cannot knowingly change its behavior between the extractor and a real execution. However, the adversarial prover does not know tdpp, so anything extracted using it might change when we switch modes.[8](#page-11-1)

To prevent this possibility, we break the proof into two parts. First, we observe that in injective mode, the existence of a preimage a[ℓ] of y[ℓ] which the CVQC verifier would accept,

Then, we use the soundness of the NP argument to establish that this holds with almost the same probability that the adversarial prover would convince the non-destructive verifier. Dodging the Cloning Bullet. As mentioned previously, at first it may seem that the ability to both extract and retain the prover's witness implies the ability to clone it, which would violate the no-cloning theorem. Now that we have explained the core of the protocol, it becomes clearer why our approach does not imply cloning.

During our proof for (knowledge) soundness, we reduce to the knowledge soundness of the original CVQC by using an injective mode fpp to extract the prover's answers for the original CVQC. However, injective mode does not have the nice recovery property we used to preserve the witness. In fact, measuring an evaluation of an injective mode fpp permanently collapses the witness. The only way to extract a witness is to take it away from the prover!

We also mention how our protocol avoids Vidick and Zhang's more formal cloning implication for proofs of knowledge [\[VZ21\]](#page-64-4). Their technique relies on the non-destructivity to query the prover on every possible verifier message and learn its classical response. The resulting table of responses acts as an inefficient clone of the prover. At that point, the extractor can be applied to each copy of the prover, extracting the prover's witness twice. In our case, the extraction and witness recovery modes have two disjoint, but indistinguishable sets of verifier messages. The prover's behavior in the witness recovery mode could indeed be cloned, but it would not be possible to extract a witness from the cloned prover. In fact, the cloned prover would be able to distinguish between an injective and recovery mode fpp simply because it does not know how to respond when the verifier sends fpp in injective mode.

### <span id="page-12-0"></span>2.2 ϵ-Repair with General Completeness.

In the more general setting of an arbitrary c, the prover's measurement in step 2 of whether they hold an accepting CVQC transcript a[ℓ] is no longer gentle. Since that measurement can noticeably disturb the state, uncomputing the trapdoor function randomness r<sup>a</sup>[ℓ] in step 3 does not allow recovering the witness.

Still, we have made some progress. In the original CVQC, the prover needed to measure the entire transcript, which can have exponentially many outcomes. In the new CVQC, the prover only makes a measurement with two outcomes, limiting the possible damage. Limiting to two possible outcomes enables a repair technique from [\[CMSZ22\]](#page-63-6).

CMSZ State Repair. Suppose one had a state |ψ⟩ which passed some verification procedure V with probability p, but then measured it with an N-outcome projective measurement. 

{13}------------------------------------------------

The measurement damages the state, resulting in |ψ ′ ⟩. Is it possible to repair |ψ ′ ⟩ so that it is accepted by V with probability close to p again?

[\[CMSZ22\]](#page-63-6) consider exactly this scenario and give an algorithm to repair |ψ ′ ⟩ with overwhelming probability. For the purposes of non-destructive CVQC, we set V to be the QMA verifier and the damaging measurement to be coherently generating a CVQC transcript and measuring whether it is accepting. If the algorithm is successful in restoring |ψ ′ ⟩'s original success probability, then the prover still has a valid witness, albeit potentially a different one.

Runtime Considerations. CMSZ's Repair algorithm is extremely useful, but has a few important limitations. First, the expected runtime scales with N, the number of outcomes for the damaging measurement. By substituting a measurement with exponentially many outcomes for a measurement with N = 2 outcomes using our approach so far, we keep the expected runtime polynomial.

Second, Repair only repairs the success probability to approximately p, plus or minus ϵ. Although ϵ can be tuned arbitrarily, the runtime also grows with 1/ϵ, so we are limited to ϵ = 1/poly(λ) if we want Repair to run in expected polynomial time. Fortunately, as discussed at the start of the overview, an ϵ = 1/poly(λ) decay suffices for our purposes.

What Can CMSZ Repair? The third and final limitation of Repair is that it can only make strong repair guarantees for a particular set of "repairable" states. If the original state |ψ⟩ was a superposition of two repairable states which were accepted with probability p<sup>1</sup> and p2, respectively, then the repair procedure will not repair to the average of p<sup>1</sup> and p2; instead, it will essentially select one at random and repair to that one. As an example, imagine that |ψ⟩ was a superposition over a "good" QMA witness which is accepted with probability 1 and some "junk" which is accepted with probability 0. If both the "good" witness and "junk" were individually repairable, then there is a possibility that Repair picks the junk state and repairs the success probability to close to 0, ruining the witness.

We show a more precise picture of which states CMSZ's Repair almost certainly will repair to an acceptance probability ≥ p ∗ , which we call "p ∗ -repairable". The overall effect of repairing a state using CMSZ's techniques can be thought of in three steps:

- 1. Estimate the probability p that |ψ⟩ passes the verification procedure V . This potentially disturbs the state.
- 2. Damage the state by measuring it.
- 3. Repair the state back to acceptance probability p, plus or minus a small error ϵ.

The primary factor determining the acceptance probability of the repaired state is the estimate in step 1. CMSZ requires a special estimation procedure that differs significantly from Marriot-Watrous's QMA verification (which can be viewed as a probability estimation 

{14}------------------------------------------------

algorithm) to make it compatible with Repair (see Section [4.1](#page-25-0) for a more in-depth discussion of this point). By closely inspecting their estimation algorithm, we observe that it approximately measures |ψ⟩ in the eigenbasis of the operator[9](#page-14-1)

$$\frac{1}{2}V + \frac{1}{4}I$$

and outputs the corresponding eigenvalue, rescaled. This operator has the same eigenstates as V with rescaled eigenvalues. As such, the set of witnesses which can be repaired almost certainly to better than p <sup>∗</sup> acceptance probability is the span of eigenstates of V with eigenvalues ≥ p <sup>∗</sup> + ϵ.

Letting a be the QMA threshold, the set of (a − ϵ)-repairable witnesses includes all Marriot-Watrous witnesses. More generally, it can be seen that any witness which is accepted by V with overwhelming probability is overwhelmingly close to being (1−negl(λ))-repairable.

### <span id="page-14-0"></span>2.3 Instantiating the Base CVQC

To obtain our main result (and to instantiate our compilers, described below), we need a "base" CVQC protocol where the honest prover uses a single copy of the QMA witness, and has arbitrary completeness c and soundness s, as long as c − s ≥ 1/poly(|x|, λ).

One way to obtain such a CVQC protocol is to use Mahadev's protocol [\[Mah18\]](#page-64-0) as the underlying CVQC protocol. This requires some care since Mahadev's CVQC consists of many repetitions of an underlying CVQC (which [\[Mah18\]](#page-64-0) constructs). As a result, the prover needs many copies of the witness, one copy for each repetition.

It is tempting to use only one copy of Mahadev's protocol, however, a single copy of this protocol has worse completeness than soundness! To demystify this, we note that what makes this protocol useful is that it has a special structure: The verifier V is defined via two verification algorithms (VT, VC); it implements VT, which is a test phase, with probability 1/2, and implements VC, which checks the validity of the witness, with probability 1/2. Importantly, V<sup>T</sup> has completeness 1 while V<sup>C</sup> has (low) completeness c. [10](#page-14-2) The guarantee is that for every x ∈ Lno and every cheating prover P ∗ that convinces V<sup>T</sup> to accept with probability close to 1, can convince V<sup>C</sup> to accept with probability at most s which is smaller than c. We denote this notion of soundness by testable soundness.

We ensure that both our compilers in Theorems [2](#page-4-0) and [3](#page-5-1) also compile CVQC protocols with testable soundness s. We refer the reader to Section [7](#page-40-0) (and in particular to Theorems [13](#page-43-0) and [14](#page-44-0) for the formal theorems).

We note that an alternative route is to use the [\[KLVY23\]](#page-63-1) compiler to obtain an underlying CVQC, as follows:

1. Take any any MIP<sup>∗</sup> protocol for L, with arbitrary completeness c and soundness s, such that c − s ≥ 1/poly(|x|, λ), that uses a single copy of a (repairable) witness.

### <span id="page-15-0"></span>2.4 Amplifying Completeness and Soundness

Let us now focus on achieving negligible errors with a single copy of the witness (while potentially allowing the witness to eventually degrade by the end of the protocol). To do this, we will have the prover and verifier sequentially repeat an ϵ-non-destructive CVQC, with completeness c and soundness s where c − s ≥ 1/poly(|x|, λ), N times, where N is a large enough polynomial in (λ, |x|,(c − s) −1 ). Such an ϵ-non-destructive CVQC can be obtained by applying the ϵ-repair technique described in Section [2.2](#page-12-0) to the base CVQC described in Section [2.3.](#page-14-0)

In our sequentially repeated protocol, the verifier will accept if and only if all the test rounds accept, and at least c+s 2 · N of the check rounds accept. If we start with an ϵ "repairable" witness that is accepted with probability p, then after N executions, with overwhelming probability we are left with a witness that is accepted with probability p − N ϵ, where we are able to set N ϵ to be an arbitrarily small inverse polynomial value. The degradation is small enough that an honest prover is still able to cause the verifier to accept in c+s 2 · N of the check rounds, except with negligible probability. This ensures negligible completeness error.

To argue that the protocol is a proof of knowledge, we will build an extractor that outputs a QMA witness for x with oracle access to any prover that generates accepting transcripts for x with noticeable probability. Our extractor will simply pick a random execution j ← [N] and run the extractor of the underlying CVQC on this execution: we will prove that this extractor succeeds with noticeable probability in finding a good QMA witness for x.

To prove correctness of this extractor, we will rely on the extractability of the underlying CVQC: namely, given any prover P ∗ that convinces V<sup>T</sup> to accept with probability close to 1, and convinces V<sup>C</sup> to accept with probability noticeably larger than s, the underlying extractor outputs a QMA witness with noticeable probability.

Then our goal is to simply prove that with noticeable probability, the session j picked by the outer extractor satisfies both the constraints above. We prove this in two parts: first, we prove that for accepting transcripts, nearly all prefixes of the transcript satisfy the following: the prover, conditioned on this prefix, will pass V<sup>T</sup> with high probability in the upcoming round. We call such a transcript a Good transcript.

Next, we prove that for nearly all Good transcripts τ , there is an i such that in session i, the prover convinces V<sup>T</sup> to accept with probability close to 1 as before, but additionally, 

{16}------------------------------------------------

passes  $V_{\rm C}$  with probability greater than s. This is proved by contradiction: suppose this were not the case, then we show that the number of check rounds that would accept will be smaller than (c+s)/2, and the transcript would be rejected by the verifier. This requires careful tail bounds for non-independent random variables, and in particular we are able to use Azuma's inequality to obtain these bounds.

The proof of soundness proceeds similarly, by noting that for any prover that outputs accepting proofs with noticeable probability, there must exist a round where the prover passes the test round with probability close to 1, and passes the check round with probability greater than s, which gives us a contradiction to the soundness of the base  $\mathsf{CVQC}$ .

This gives us a protocol that uses a single copy of the witness, and achieves negligible soundness and completeness errors, albeit at the cost of slightly degrading the witness. We can then apply the compiler described in Section 2.1 again to the resulting protocol to make it witness preserving (i.e. with negligible disturbance to the witness state).

#### <span id="page-21-0"></span>3.6 Classical Verification of Quantum Computation

<span id="page-21-1"></span>**Definition 6.** A Classical Verification of Quantum Computation (CVQC) protocol for a  $QMA_{a,b}$  language  $\mathcal{L} = (\mathcal{L}_{yes}, \mathcal{L}_{no})$  associated with a verifier  $V_{QMA} = \{V_{QMA,x}\}_{x \in \mathcal{L}_{yes} \cup \mathcal{L}_{no}}$  deciding  $\mathcal{L} \in QMA_{a,b}$ , is an interactive protocol between a quantum prover P and a classical verifier V.

It satisfies the following for some parameters  $c = c(\lambda, |x|)$  and  $s = s(\lambda, |x|)$ .

• Completeness c. For every  $\lambda \in \mathbb{N}$ , every  $x \in \mathcal{L}_{yes}$  of size at most  $2^{\lambda}$ , and every witness  $|w\rangle$  such that

$$\Pr[V_{\mathsf{QMA},x}(|w\rangle) = 1] \ge a,$$

it holds that

$$\Pr[(P(|w\rangle), V)(1^{\lambda}, x) = 1] \ge c(\lambda, |x|).$$

Remark 2. Many of the CVQC protocols in the literature (such as [Mah18]) need many copies of the witness to obtain completeness. The focus of this work is on constructing CVQC protocols where the prover uses only a single copy of the witness, and hence we define completeness in this restrictive manner.

• Computational soundness s. For every QPT cheating prover  $P^*$ , every  $\lambda \in \mathbb{N}$  and every  $x \in \mathcal{L}_{no}$  it holds that for for every state  $|\psi\rangle$ ,

$$\Pr[P^*(|\psi\rangle), V)(1^{\lambda}, x) = 1] \le s(\lambda, |x|).$$

• Efficiency. For every  $\lambda$  and every  $x \in \mathcal{L}_{yes}$  with a corresponding witness  $|w\rangle$ , the honest prover  $P(1^{\lambda}, x, |w\rangle)$  is a (uniform) quantum circuit of size  $poly(\lambda, |x|)$ , and the verifier  $V(1^{\lambda}, x)$  is a classical probabilistic polynomial time circuit of size  $poly(\lambda, |x|)$ .

A CVQC protocol can additionally be a classical proof of quantum knowledge [\[VZ21\]](#page-64-4), which is a generalization of the classical concept of a proof of knowledge to the quantum setting.

<span id="page-22-1"></span>Definition 7 (η-Argument of Knowledge). Let R be a QPT algorithm outputting a decision bit, which we call the "QMA relation". An interactive protocol between a classical verifier and a quantum prover is an η-argument of (quantum) knowledge for R if there exists a QPT extractor E such that for every statement x ∈ Lyes ∪ Lno and every dishonest QPT prover P ∗ which convinces the verifier with probability s(λ, |x|), the extractor outputs a quantum state satisfying

> Pr[Accept ← R(1<sup>λ</sup> , x, ρ) : ρ ← E(1<sup>λ</sup> , P<sup>∗</sup> , x)] ≥ η(λ, |x|)s(λ, |x|).

This definition is weaker than the one in the classical setting, which guarantees that if P ∗ convinces the verifier w.p. <sup>1</sup> p(λ) then the extractor extracts a valid witness with overwhelming probability (after running in time poly(λ, p(λ)). This difference is inherent since if P <sup>∗</sup> has a state α |w⟩ + β |junk⟩ then we cannot hope to extract |w⟩ with probability greater than |α| 2 .

Testable Soundness and Proof of Knowledge. For use in our technical sections, we now define testable soundness and proof of knowledge. Intuitively, we amplify by sequentially repeating a base CVQC protocol with high completeness and soundness errors, and indeed where the completeness error is higher than the soundness error. Fortunately, the usefulness of this protocol comes from a special structure: the verifier V is specified by a pair of verification algorithms (VT, VC). On any execution, the verifier chooses between them uniformly at random: it runs a test phase V<sup>T</sup> with probability 1/2 and a check phase V<sup>C</sup> with probability 1/2. Crucially, V<sup>T</sup> has completeness 1, whereas V<sup>C</sup> has a lower completeness c [14](#page-22-0) .

The protocol's security guarantee is as follows: for every x ∈ Lno and any cheating prover P ∗ that succeeds at convincing V<sup>T</sup> with probability close to 1, the probability of convincing V<sup>C</sup> is bounded above by s < c. We define testable soundness below, parameterized by s—the upper bound on the probability that V<sup>C</sup> accepts x ∈ Lno.

<span id="page-22-2"></span>Definition 8 (s-Testable Soundness). A CVQC protocol for language L has testable soundness s = s(λ, |x|) if there is a constant ζ > 0 such that the following holds. The verifier uniformly randomly picks one out of two verification algorithms: a "test" algorithm V<sup>T</sup> and a "check" algorithm VC, where V<sup>T</sup> has perfect completeness, and for every QPT cheating prover P ∗ , every λ ∈ N and every x ∈ Lno it holds that for for every (mixed) state ρ consisting of at most poly(λ, |x|) qubits, if

$$\Pr[P^*(\boldsymbol{\rho}), V_{\mathsf{T}})(1^{\lambda}, x) = 1] \ge \zeta \tag{2}$$

then

$$\Pr[P^*(\boldsymbol{\rho}), V_{\mathsf{C}})(1^{\lambda}, x) = 1] \le s(\lambda, |x|) \tag{3}$$

### <span id="page-24-0"></span>4 Repairing High-Quality Witnesses

In this section, we prove that [CMSZ22]'s state repair procedure can repair high quality QMA witnesses – those that are accepted with overwhelming probability – with overwhelming probability.

In particular, [CMSZ22]'s state repair procedure uses a special probability estimation algorithm ValEst.<sup>16</sup> Their repair procedure can be thought of as using ValEst to estimate the probability p of  $V(|\psi\rangle)$  accepting, then repairing a damaged  $|\psi\rangle$  so that the repaired state is accepted with probability  $p-\epsilon$ . However, ValEst is only guaranteed to be accurate on average. For example, if  $|\psi\rangle$  has success probability p, and is a superposition of  $|\psi_1\rangle$  and  $|\psi_2\rangle$ , where  $|\psi_1\rangle$  is accepted with probability 1 and  $|\psi_2\rangle$  is accepted with probability 0, the ValEst procedure may collapse  $|\psi\rangle$  into a mixture of  $|\psi_1\rangle$  and  $|\psi_2\rangle$ . If the witness is collapsed to  $|\psi_2\rangle$ , then the repair procedure will never return the success probability to  $\approx p$ . Although intuitively an overwhelmingly-good witness cannot collapse like this, that fact surprisingly does not follow from the black-box properties of ValEst presented in [CMSZ22].<sup>17</sup>

To ensure that we can repair a QMA witness  $|\psi\rangle$  for QMA, we need to ensure that  $\mathsf{ValEst}(|\psi\rangle)$  gives an estimate close to the probability that  $|\psi\rangle$  passes the QMA verification procedure (e.g. 2/3) with overwhelming probability. This will rule out the possibility of a problematic collapse, except with negligible probability.

We show in this section that if  $|\psi\rangle$  is an eigenvector of the verification procedure V with eigenvalue p, then  $\mathsf{ValEst}(|\psi\rangle)$  returns an estimate  $p\pm\epsilon$  with overwhelming probability. Such states are exactly Marriot-Watrous witnesses when p is greater than the QMA threshold (e.g. 2/3). As a corollary, any damaged witness which was originally accepted by V with overwhelming probability can be repaired to a state which is accepted with  $1-3\epsilon-\mathsf{negl}(\lambda)$  probability.<sup>18</sup>

If this is the case, the first statement of the lemma holds immediately. The second statement, for  $|\phi\rangle = \sum_j \alpha_j |\psi_j\rangle$ , follows from the observation that a j such that  $p_j \in [p_1^*, p_2^*]$  will be sampled with probability  $\geq 1 - \eta$ .

By Lemma 3, it in turn suffices to show that  $|\psi\rangle \otimes |0,+_{\perp,\perp}\rangle$  is an eigenstate of

$$\left(I_{\mathcal{H}}\otimes|0,+_{\top,\perp}\rangle\langle0,+_{\top,\perp}|\right)\Pi_{G}\left(I_{\mathcal{H}}\otimes|0,+_{\top,\perp}\rangle\langle0,+_{\top,\perp}|\right)$$

with eigenvalue  $(p^*/2+1/4)$ . The eigenstates of this operator are the same as the eigenstates of

$$\left(I\otimes\langle 0,+_{\top,\perp}|\right)\Pi_{G}\bigg(I_{\mathcal{H}}\otimes|0,+_{\top,\perp}\rangle\bigg)$$

with the same eigenvalues, when the latter are appended with  $|0,+_{\top,\perp}\rangle$ . Expanding this operator,

$$\begin{split} \left(I_{\mathcal{H}} \otimes \langle 0, +_{\top, \bot}|\right) \Pi_{G} \bigg(I_{\mathcal{H}} \otimes |0, +_{\top, \bot}\rangle \bigg) &= \frac{1}{2} \bigg(I_{\mathcal{H}} \otimes \langle 0, 0|\bigg) U_{V}^{\dagger} \Pi_{V}^{\mathsf{Accept}} U_{V} \bigg(I_{\mathcal{H}} \otimes |0, 0\rangle\bigg) \\ &+ \frac{1}{2} \bigg(I_{\mathcal{H}} \otimes \langle 0, 0|\bigg) \Pi_{G} \bigg(I_{\mathcal{H}} \otimes |0, \top\rangle\bigg) \\ &+ \frac{1}{2} \bigg(I_{\mathcal{H}} \otimes \langle 0, \top|\bigg) \Pi_{G} \bigg(I_{\mathcal{H}} \otimes |0, 0\rangle\bigg) \\ &+ \frac{1}{4} \bigg(I_{\mathcal{H}} \otimes \langle 0, \top|\bigg) I\bigg(I_{\mathcal{H}} \otimes |0, \top\rangle\bigg) \end{split}$$

Since  $\Pi_G$  is diagonal on  $|0\rangle_{\mathcal{T}}$ ,  $|\top\rangle_{\mathcal{T}}$ , and  $|\bot\rangle_{\mathcal{T}}$  the cross terms cancel and we are left with

$$\frac{1}{2}V + \frac{1}{4}I_{\mathcal{H}}$$

# <span id="page-29-0"></span>5 Dual-Mode Trapdoor Functions with State Recovery.

In this section we define the notion of a dual-mode trapdoor function family with state recovery. Intuitively, this is a family of (randomized) functions that have two modes: an injective mode and a recovery mode, and it is hard to distinguish between the two. Functions in the injective mode are injective and are associated with a trapdoor which can be used to invert the function efficiently. Functions in the recovery mode are also associated with a trapdoor, where this trapdoor is used to recover the state (this roughly corresponds to the lossy/twoto-one mode in the existing literature on dual-mode TCFs). More specifically, functions in recovery mode have the property that if we compute the function coherently on a quantum state and measure the output value then this does not disturb the state, and we can use the trapdoor to recover the quantum state.

<span id="page-29-2"></span>Definition 11 (Dual-Mode Trapdoor Function Family with State Recovery.). A dual-mode trapdoor function family with state recovery is a family of (potentially randomized) functions

$$\mathcal{F} = \{f_{\mathsf{pp}} : \mathcal{X} \times \mathcal{R} \to \mathcal{Y}\}_{\mathsf{pp}}$$

satisfying the following properties.[21](#page-29-1)

• Efficient Generation. There exists a probabilistic poly-time algorithm Setup distribution, which takes as input security parameter 1 <sup>λ</sup> along with a flag mode ∈ {recovery, injective}, and outputs

$$(\mathsf{pp},\mathsf{td}) \leftarrow \mathsf{Setup}(1^{\lambda},\mathsf{mode}).$$

<span id="page-29-1"></span><sup>21</sup>Formally, X , R, Y all depend on the security parameter with respect to which pp was generated. This is omitted from the notation for the sake of simplicity.

{30}------------------------------------------------

• Dual-Mode Indistinguishability.

$$\{\mathsf{pp}: (\mathsf{pp},\mathsf{td}) \leftarrow \mathsf{Setup}(1^{\lambda},\mathsf{injective})\} \approx_c \{\mathsf{pp}: (\mathsf{pp},\mathsf{td}) \leftarrow \mathsf{Setup}(1^{\lambda},\mathsf{recovery})\}$$
  
where  $by \approx_c we\ mean\ computational\ indistinguishability\ against\ \mathsf{QPT}\ algorithms.$ 

- Efficient Evaluation and Range Superposition. There is a poly-time computable evaluation function that given any pp, in the image of Setup and given any pair (x, r) ∈ (X × R) outputs fpp(x, r).
  - Furthermore, fpp is associated with a distribution over randomness R with probability mass function p such that the state |randpp⟩ = P <sup>r</sup>∈R p p(r)|r⟩ is efficiently preparable. This state is a superposition over randomness used to evaluate fpp.
- Injective Mode: Trapdoor. There exists a poly-time algorithm Ext such that for all (pp,td) ∈ SUPP(Setup(1<sup>λ</sup> , injective)) and all input/randomness pairs (x, r) ∈ X × R,

$$\Pr[x \leftarrow \mathsf{Ext}(\mathsf{td}, f_{\mathsf{pp}}(x; r))] = 1$$

- Recovery Mode: Trapdoor. Consider the following experiment Exp(pp, |ψ⟩), parameterized by a public parameter pp and a quantum state |ψ⟩ = P <sup>x</sup> α<sup>x</sup> |ϕx⟩<sup>A</sup> ⊗ |x⟩<sup>B</sup> in registers A and B.

  - 2. Measure the register containing fpp(x; r) in the computational basis to obtain y. Let |φy⟩ be the residual state.

$$\Pr\left[\frac{1}{2}\big\|\,|\psi\rangle\langle\psi|-\rho\big\|_1 \leq \mathsf{negl}(\lambda): \begin{array}{c} (|\varphi_y\rangle\,,y) \leftarrow \mathsf{Exp}(\mathsf{pp},|\psi\rangle) \\ \rho \leftarrow \mathsf{Recover}(\mathsf{td},|\varphi_y\rangle\,,y) \end{array}\right] = 1 - \mathsf{negl}(\lambda)$$

$$\mathsf{Recover}_{\mathsf{td}} \triangleq \mathsf{Recover}(\mathsf{td}, \cdot, \cdot) \quad \mathit{and} \quad \mathsf{Exp}_{\mathsf{pp}} \triangleq \mathsf{Exp}(\mathsf{pp}, \cdot).$$

As a more granular definition, if the recovered state ρ is has trace distance δ to the original state |w⟩, we refer to the function family as having (1 − δ)-state recovery.

<span id="page-36-0"></span>6 State-Preserving Arguments for NP

In this section, we construct an interactive argument for NP with the property that if the (honest) prover starts with a state  $|\psi\rangle = \sum_{w} \alpha_w |w\rangle$  which is a superposition over valid witnesses, then after running the protocol it is left with a state  $\rho$  that is negligibly far from its original state  $|\psi\rangle$ . <span id="page-36-2"></span>**Definition 12.** A state-preserving argument for an NP language  $\mathcal{L}$  is an interactive protocol between a QPT prover P and a PPT verifier V that satisfies the following guarantees.

• Completeness and State Preservation. For every  $x \in \mathcal{L}$  and every superposition  $|\psi\rangle = \sum_{w} \alpha_{w} |w\rangle_{\mathcal{W}} \otimes |\phi_{w}\rangle_{\mathcal{B}}$  over witnesses for x (encoded in the computational basis and potentially entangled with an external state  $|\phi_{w}\rangle$ ), there exist negligible functions  $\mu_{1}$  and  $\mu_{2}$  such that for every  $\lambda \in \mathbb{N}$ ,

$$\Pr\left[\left(\frac{1}{2}\big\|\,|\psi\rangle\langle\psi|-\rho\big\|_1\leq\mu_1(\lambda)\right)\quad\wedge\quad(b=\mathsf{Accept}):\ (\rho,b)\leftarrow\langle P(|\psi\rangle),V\rangle(1^\lambda,x)\right]\geq 1-\mu_2(\lambda),$$

where P acts only on register W and where the notation  $(\rho, b) \leftarrow \langle P(|\psi\rangle), V \rangle (1^{\lambda}, x)$  means that V outputs  $b \in \{Accept, Reject\}$  and P's residual state is  $\rho$ .

When we wish to be more precise about the repair guarantees, we say that the argument is  $\mu_1$ -state-preserving.

• Computational Soundness. For every polynomial  $\ell = \ell(\lambda)$ , there exists a negligible function  $\mu$  such that for every  $\lambda \in \mathbb{N}$ , every  $x \notin \mathcal{L}$  of size  $\leq \ell(\lambda)$  and every QPT cheating prover  $P^*$  with auxiliary input  $|\psi\rangle$  consisting of at most  $\mathsf{poly}(\ell(\lambda))$  qubits

$$\Pr[\langle P^*(|\psi\rangle, V\rangle(1^{\lambda}, x) = \mathsf{Accept}] \le \mu(\lambda)$$

We say a state-preserving argument for NP is an  $\delta$ -argument of knowledge if there exists a QPT extractor Ext such that for every QPT adversarial prover  $P^*$  and every statement x,

$$\Pr[\mathsf{Accept} \leftarrow \langle P^*, V \rangle (1^{\lambda}, x)] \leq \Pr[(x, w) \in \mathcal{R}_{\mathcal{L}} : w \leftarrow \mathsf{Ext}(P^*)] + \delta$$

where  $\mathcal{R}$  is the NP relation for  $\mathcal{L}$ . If  $\delta = \text{negl}(\lambda)$ , we simply say that it is an argument of knowledge.

<span id="page-36-1"></span>State preservation composes in parallel since it also preserves entanglement with an external register. As a result, state-preserving arguments can be parallely amplified whenever parallel repetition amplifies soundness: coherently classical-copy the witness to produce  $\sum_{w} \alpha_{w} |w\rangle^{\otimes n}$  and independently run a state-preserving argument on each witness copy.

Lemma 9. The n-fold parallel repetition of a (1 − δ)-state-preserving argument is (1 − nδ) state preserving.

Proof. Since each repetition acts on disjoint registers from the others, the repetitions commute (for the purposes of determining the leftover state). Therefore we may consider running the repetitions one at a time. Let ρ<sup>i</sup> be the leftover state after repetition i, with ρ<sup>0</sup> = |ψ⟩⟨ψ| being the initial witness superposition. By state-preservation and union bound, 1 2 ∥ρ<sup>i</sup> − ρi+1∥<sup>1</sup> ≤ δ for every i with overwhelming probability. By triangle inequality the leftover state has distance ≤ nδ from the prover's initial input |ψ⟩⟨ψ| with overwhelming probability.

### <span id="page-37-0"></span>6.1 Construction

We construct witness preserving arguments for NP by building on top of the well-known witness indistinguishable argument for the NP-complete 3-coloring language. Instances for 3-coloring are graphs x = (V, E). Witnesses are a 3-coloring of the graph, i.e., a function w : V → [3], such that for every edge (i, j) ∈ E, the vertices i and j have different colors: w(i) ̸= w(j). We consider w to be written as a truth-table and denote the color it assigns to vertex i by w<sup>i</sup> .

Our construction makes use of a dual-mode trapdoor function family F, with a corresponding setup algorithm, denoted by Setup<sup>F</sup> (see Definition [11\)](#page-29-2). We describe a version with 1/poly(λ) soundness. Soundness can be amplified by running the protocol sequentially, without losing state preservation.

<span id="page-37-1"></span>Construction 2 (State-Preserving Arguments for NP). Let L be the 3-coloring language and consider any statement x ∈ L. The prover starts with a superposition P <sup>w</sup> α<sup>w</sup> |w⟩ over witnesses w for x ∈ L.

During the protocol, the prover will maintain three registers W, P, and R.

- W = (W1, . . . , W|V|) will contain the witness and is initialized to P <sup>w</sup> α<sup>w</sup> |w⟩.
- P will contain a permutation π : [3] → [3] and is initialized to ∝ P <sup>π</sup>∈Perm([3]) |π⟩.
- R = (R1, . . . , R|V|) will contain randomness r<sup>i</sup> for evaluating a trapdoor function and is initialized to N <sup>i</sup>∈V |randpp⟩.
- 1. Verifier: Sample (pp,td) ← Setup<sup>F</sup> (1<sup>λ</sup> , recovery). Send pp to the prover.

#### 2. Prover:

(a) For every i ∈ V apply an isometry

$$|w_i, \pi, r_i\rangle_{\mathcal{W}_i, \mathcal{P}, \mathcal{R}_i} \mapsto |w_i, \pi, r_i\rangle_{\mathcal{W}_i, \mathcal{P}, \mathcal{R}_i} \otimes |f_{\mathsf{pp}}(\pi(w_i); r_i)\rangle$$

and measure the last register to obtain y<sup>i</sup> .

- (b) Send  $y_{\mathcal{V}} := (y_1, \dots, y_{|\mathcal{V}|})$  to the verifier.
- 3. Verifier: Sample a random edge  $(i, j) \leftarrow \mathcal{E}$  and send (i, j) to the prover.
- 4. **Prover:** Measure  $(\pi(w_i), \pi(w_j), r_i, r_j)$  using registers  $\mathcal{P}$ ,  $\mathcal{W}_i$ ,  $\mathcal{W}_j$ ,  $\mathcal{R}_i$ , and  $\mathcal{R}_j$ , and send it to the verifier.
- 5. **Verifier:** Upon receiving  $c = (c_i, c_j, r_i, r_j)$  from the prover, check that  $c_i \neq c_j$ , that  $y_i = f_{pp}(c_i; r_i)$ , and that  $y_j = f_{pp}(c_j; r_j)$ . If these are all true, output Accept and send td to the prover. Otherwise, output Reject.
- 6. **Prover:** For each  $k \in \mathcal{V} \setminus \{i, j\}$ , run Recover<sub>\mathcal{F}</sub>(td, \cdot, y\_k) on registers  $(\mathcal{W}_k, \mathcal{P}_k)$ .

Next, observe that for every witness w, there is a unique permutation  $\pi_{c,w,(i,j)}$  such that  $c_i = \pi_{c,w}(w_i)$  and  $c_j = \pi_{c,w}(w_j)$ ; this holds because  $w_i \neq w_j$  and fixing two outputs of a permutation on [3] fully determines the permutation. Compute the unitary swapping

$$|w, \pi_{c,w,(i,j)}\rangle_{\mathcal{WP}} \leftrightarrow |w, \mathsf{id}\rangle_{\mathcal{WP}}$$

where id is the identity permutation, and acting as the identity elsewhere. Output register  $\mathcal{W}$ .

*Proof.* We need to prove that the completeness, soundness and state preservation properties hold.

**Completeness.** Completeness follows from the observation that  $\pi(w_i) \neq \pi(w_j)$  for all valid witnesses w.

Soundness and Argument of Knowledge. It suffices to prove soundness in the hybrid experiment where in the first round the verifier sends pp which are generated in injective mode:  $(pp, td) \leftarrow Setup_{\mathcal{F}}(1^{\lambda}, injective)$ , as opposed to lossy mode. The dual-mode indistinguishability property implies that this experiment is indistinguishable from the real execution. Therefore, for any QPT prover  $P^*$ , its success probability changes only by a negligible factor.

In injective mode, each y<sup>i</sup> corresponds to a unique preimage color c<sup>i</sup> . If x /∈ L, then there must exist an edge (i, j) ∈ E such that c<sup>i</sup> = c<sup>j</sup> or such that c<sup>i</sup> ∈/ [3]. Therefore, P ∗ 's success probability is at most 1−1/|E| in the hybrid experiment, and thus at most 1−1/|E|−negl(λ) in a real execution.

The argument of knowledge extractor Ext interacts with P <sup>∗</sup> using an injective mode pp until it receives y<sup>V</sup> for each instance. Then, it inverts each y<sup>V</sup> using the trapdoor to obtain a coloring c<sup>V</sup> for each instance. It checks which coloring is valid and outputs that coloring. As before, P ∗ 's success probability is unmodified by this extractor. $$\propto \sum_{w} \alpha_{w} |w\rangle \otimes \sum_{\pi \in \mathsf{Perm}([3])} |\pi\rangle \otimes \bigotimes_{i \in V} \sum_{r_{i} \in \mathcal{R}} \beta_{r_{i}} |r_{i}\rangle$$

where β<sup>r</sup><sup>i</sup> is an amplitude determined by F and pp. Suppose that the prover (with the verifier's aid) ran all computations on k ∈ V\{i, j} before doing any computation on registers corresponding to i and j. Explicitly, they apply the isometry from step 2, measure yk, and run Recover<sup>F</sup> (td, ·, yk) before doing any operations to the registers corresponding to i and j. Since these computations are done on disjoint registers, reordering them produces an identical state.

The trapdoor property of F in recovery mode implies that the state immediately after operating on all k ∈ V\{i, j}, before operating on i and j, has trace distance (|V|−2)2<sup>−</sup><sup>λ</sup>/|V| < 2 −λ from

$$\propto \sum_{w} \alpha_{w} \left| w \right\rangle \otimes \sum_{\pi \in \mathsf{Perm}([3])} \left| \pi \right\rangle \otimes \sum_{r_{i}, r_{j} \in \mathcal{R}} \beta_{r_{i}} \beta_{r_{j}} \left| r_{i}, r_{j} \right\rangle$$

We will now rewrite this state in terms of the prover's possible last messages – specifically, the revealed colors c = (c<sup>i</sup> , c<sup>j</sup> ). Fix any c and any witness w. The possible settings of π are constrained by π(wi) = c<sup>i</sup> and π(w<sup>j</sup> ) = c<sup>j</sup> . Since w<sup>i</sup> ̸= w<sup>j</sup> for valid witnesses, this leaves only one possibility for π to map the last color. In other words, there is a unique πc,w,(i,j) which is consistent with c, w, and (i, j). Thus, we can rewrite the state as

$$\propto \sum_{w} \alpha_{w} |w\rangle \otimes \sum_{c} |\pi_{c,w,(i,j)}\rangle \otimes \sum_{r_{i},r_{j} \in \mathcal{R}} \beta_{r_{i}} \beta_{r_{j}} |r_{i},r_{j}\rangle$$

Measuring c, r<sup>i</sup> , r<sup>j</sup> , and the evaluations of fpp on indices i and j collapses the state to

$$\sum_{w} \alpha_w |w\rangle \otimes |\pi_{c,w,(i,j)}\rangle$$

Finally, applying a unitary swapping

$$|w, \pi_{c,w,(i,j)}\rangle \leftrightarrow |w, \mathsf{id}\rangle$$

and discarding the register containing |id⟩ recovers the original state P <sup>w</sup> α<sup>w</sup> |w⟩, up to 2<sup>−</sup><sup>λ</sup> trace distance.

# <span id="page-40-0"></span>7 Non-Destructive CVQC for QMA

In this section we construct a CVQC for any QMA language with nearly perfect completeness and soundness, while using only a single copy of the QMA witness, and moreover the protocol is witness preserving (as formally defined below).

Clearly, we cannot hope to obtain negligible soundness with any witness, since the witness can be a superposition of a "good" witness and "junk." Indeed, we define the set of "repairable" witnesses for every x ∈ Lyes, and argue that the honest prover only needs a single copy of a "repairable" witness, to convince the verifier that x ∈ Lyes with overwhelming probability.

The set of repairable witnesses includes every witness |w⟩ that is accepted by VQMA,x with overwhelming probability, as well as all of the Marriot-Watrous witnesses. So, as a special case, we obtain the following theorem.

The end result is an ϵN-non-destructive CVQC protocol with completeness 1 − negl(λ) and soundness negl(λ). Thus, after running the entire protocol, we are left with a witness that remains in Repairablep−ϵN,ϵ(x) for some p ≥ a + ϵN.

Note that if we started with a protocol that is non-destructive (i.e., ϵ-non-destructive for ϵ = negl(λ)) then the resulting protocol is also non-destructive. For the general completeness case, Theorem [13](#page-43-0) is restricted to ϵ = 1/poly(λ) due to poly(1/ϵ) runtime increase.

### <span id="page-56-0"></span>7.5 Proof of Theorem [14](#page-44-0) (Sequential Repetition)

In this section we prove Theorem [14.](#page-44-0) Namely, we construct a compiler that converts any ϵ-non-destructive CVQC into one that has negligible soundness and almost perfect completeness, while still being ϵ-non-destructive (as defined in Theorem [14\)](#page-44-0). The compiler consists of many sequential repetitions of the underlying CVQC protocol, and the verifier accepts if the number of accepting executions exceeds a threshold.

Construction 6 (Sequential Amplification of ϵ-Non-Destructive CVQC). Fix a language L = (Lyes,Lno) ∈ QMAa,b with a QMA verifier VQMA = {VQMA,x}x∈Lyes∪Lno . Let (P, V ) be an ϵ-non-destructive CVQC for L w.r.t. VQMA, with completeness c and testable soundness s, where c − s ≥ 1/poly(λ), and where V = (VT, Vcheck).

For N large enough (that we set below), we construct an N ϵ-non-destructive CVQC protocol (P ′ , V ′ ) for L w.r.t. VQMA with negligible soundness and overwhelming completeness. P ′ and V ′ take as input (1<sup>λ</sup> , x) where x ∈ Lyes, and in P ′ takes as an additional input a witness |w0⟩ ∈ Repairablep,ϵ(x). The protocol (P ′ (|w0⟩), V )(1<sup>λ</sup> , x) proceeds as follows:

- 1. Set p<sup>0</sup> = 1 − 2 −λ .
- 2. Set N = |x| <sup>d</sup>λ d+2 (c−s) 2 .
- 3. For i = 1, . . . , N, do the following:

Emulate the protocol P |wi−1⟩ , V (1<sup>λ</sup> , x), where V implements V<sup>T</sup> with probability 1/2 and implements Vcheck with probability 1/2.

- (a) If V implemented V<sup>T</sup> and the test failed (i.e., V<sup>T</sup> rejects), then V ′ aborts and outputs Reject.
- (b) If V implemented Vcheck then denote by b<sup>i</sup> the output bit of V (1 denotes accept, 0 denotes reject).
- (c) Denote by |wi⟩ the residual state of P.
- 4. Denote by B ⊆ [N] all the repetitions where V executed Vcheck.
- 5. V ′ outputs Accept if P i∈B b<sup>i</sup> ≥ c+s 2 |B|. Otherwise, V ′ outputs Reject.
- 6. P ′ outputs |w<sup>N</sup> ⟩.

Proof. We prove that (P ′ , V ′ ) is ϵN-non-destructive, and that it has 1−negl(λ) completeness and negl(λ) computational soundness error. Below, we assume that |x| > λ, which can be achieved without loss of generality by padding the instance.

**Argument of Knowledge.** Fix any  $x \in \mathcal{L}_{yes} \cup \mathcal{L}_{no}$ , and any efficient prover strategy  $P^*$  with arbitrary initial quantum advice  $\rho^*$  such that the verifier in the sequentially repeated protocol accepts with noticeable probability  $\frac{1}{p_{acc}(\lambda)}$ .

Our goal is to build a proof-of-knowledge extractor for the repeated protocol. Our extractor is inspired by [VZ21]. Looking ahead, our extractor will simply pick a random execution  $j \leftarrow [N]$  and run the extractor of the underlying CVQC on this execution. We show that such an extractor succeeds with probability  $\frac{\delta}{p_{\text{acc}}N}(\lambda) - \text{negl}(\lambda)$ , where  $\delta(\lambda)$  is the success probability of the underlying CVQC extractor.

Formally, we will first prove that for accepting transcripts, nearly all prefixes of the transcript satisfy the following: the prover, conditioned on this prefix, will pass  $V_{\mathsf{T}}$  with high probability in the upcoming round. We call such a transcript a Good transcript.

**Definition 16** (Good Transcripts). $$\Pr\left[P_i^*(\rho_{i-1}), V_{i,\mathsf{T}}(1^{\lambda}, x) = 1 \middle| \tau_{i-1} \right] > \left(1 - \frac{1}{\lambda^d}\right)$$

The next claim proves that an overwhelming fraction of accepting transcripts are Good.

<span id="page-58-1"></span>Claim 7. There exists a negligible function  $\mu(\cdot)$  such that

$$\Pr[\tau_{\lambda} \in \mathsf{Good}_{\lambda} | \tau_{\lambda} \text{ is accepted}] \geq (1 - \mu(\lambda))$$

$$\Pr[\tau_{\lambda} \not\in \mathsf{Good}_{\lambda} | \tau_{\lambda} \text{ is accepted}] \geq \frac{1}{\mathsf{poly}(\lambda)}$$

$$\Pr\left[P_i^*(\rho_{i-1}), V_{i,\mathsf{T},\mathsf{NDCVQC}}(1^{\lambda}, x) = (\cdot, 1) \middle| \tau_{i-1} \right] < \left(1 - \frac{1}{\lambda^d}\right)$$

Since the verifier picks "test" with constant probability, such accepting transcripts are only generated with probability at most  $\left(1 - \frac{1}{\lambda^d}\right)^{\lambda^{d+1}} = \mathsf{negl}(\lambda)$ , a contradiction.

and therefore this noticeable fraction of Good transcripts  $\tau$  are not accepted, a contradiction.

Combining Claim 7 and Claim 9, we have that with probability  $(1 - \mathsf{negl}(\lambda))$  over  $\tau_{\lambda} \in \mathsf{Good}_{\lambda}$ , there exists  $j \in [N]$  such that:

$$\Pr\left[P_j^*(\rho_{j-1}), V_{j,\mathsf{T},\mathsf{NDCVQC}}(1^{\lambda}, x) = (\cdot, 1) \middle| \tau_{j-1} \right] > \left(1 - \frac{1}{\lambda^d}\right)$$

and

$$\Pr\left[P_{j}^{*}(\rho_{j-1}), V_{j,\mathsf{C},\mathsf{NDCVQC}}(1^{\lambda}, x) = (\cdot, 1) | \tau_{j-1}\right] > s + \frac{1}{p(\lambda)}$$

and furthermore, the event  $\tau_{\lambda} \in \mathsf{Good}_{\lambda}$  happens with probability  $\frac{1}{p(\lambda)} - \mathsf{negl}(\lambda)$ .

<span id="page-60-0"></span>Thus, an extractor that randomly picks a sequential round  $j \leftarrow [N]$  and runs the extraction procedure of the underlying CVQC on round j will, with probability  $\frac{1}{pN}(\lambda) - \mathsf{negl}(\lambda)$ , sample j satisfying the two conditions above, and therefore will output a witness with probability  $\frac{\delta}{pN}(\lambda) - \mathsf{negl}(\lambda)$ , where  $\delta(\lambda)$  is the (non-negligible) probability that the underlying extractor succeeds.

{61}------------------------------------------------

**Remark 4.** The factor N loss is difficult to eliminate due to the following case: the adversary participates in  $\frac{c+s}{2}N$  executions with an extremely good witness that the CVQC accepts with probability 1, and uses a junk state in the remaining executions. In this case, the extractor would need to identify an execution where the adversary uses a good witness.

**Soundness.** Fix any  $x \in \mathcal{L}_{no}$  and any prover  $P^*$  that generates accepting proofs for x with probability  $\frac{1}{p_{acc}(\lambda)}$ , for some polynomial  $p_{acc}$ . Towards a contraction, for such a prover, note that by Claim 7 and Claim 9 above, there exists  $j \in [N]$ , transcript prefix  $\tau_{j-1}$  and corresponding prover state  $\rho_{j-1}$  such that:

$$\Pr\left[P_j^*(\rho_{j-1}), V_{j,\mathsf{T},\mathsf{NDCVQC}}(1^{\lambda}, x) = (\cdot, 1) \middle| \tau_{j-1} \right] > \left(1 - \frac{1}{\lambda^d}\right)$$

and

$$\Pr\left[P_{j}^{*}(\rho_{j-1}), V_{j,\mathsf{C},\mathsf{NDCVQC}}(1^{\lambda}, x) = (\cdot, 1) | \tau_{j-1}\right] > s + \frac{1}{p(\lambda)}$$

This circuit  $P_j^*$  with input state  $\rho_{j-1}$  contradicts testable s-soundness of the underlying CVQC, as desired.

# <span id="page-65-0"></span>A Proofs of No-Intrusion

As an application of state-preserving arguments for NP, we show how to construct public-key encryption (PKE) with proofs of no-intrusion (PoNIs). PKE with PoNIs was introduced by [\[GR25\]](#page-63-3). Roughly, it allows a classical verifier to test that a quantum ciphertext has not been stolen from a server, in the sense that nobody besides the server can decrypt it even given the key, without destroying the server's copy of the ciphertext.

Our construction achieves a stronger notion of verifiability than [\[GR25\]](#page-63-3). In their construction, only the original encryptor can verify their own ciphertext because the PoNI verifier requires secret information about it. In our construction, the encryptor may publish part of their verification key to allow anyone to act as the PoNI verifier.

### <span id="page-65-1"></span>A.1 Definition

Definition 17 (Encryption with Proofs of No-Intrusion). A (public-key) encryption scheme with a proof of no-intrusion consists of the following QPT algorithms.

- (pk,sk) ← KeyGen(1<sup>λ</sup> ) takes as input the security parameter 1 λ then outputs a public key pk and a secret key sk.
- vk ← VKGen(1<sup>λ</sup> ) takes as input the security parameter 1 λ then outputs a verification key vk.
- ct ← Enc(pk, m, vk) takes as input a public key pk, a message m, and a verification key vk, then outputs a ciphertext ct.
- m ← Dec(sk, ct) takes as input a secret key sk and a ciphertext ct, then outputs a message m.

Additionally, it is equipped with a proof of no-intrusion, which is an interactive protocol with classical communication between a QPT prover P holding a state |ψ⟩ and a PPT verifier holding a verification key vk:

$$(\ket{\psi'}, b) \leftarrow \mathsf{PoNI}\langle \mathsf{P}(\ket{\psi}), \mathsf{V}(\mathsf{vk}) \rangle$$

At the end of the protocol, P outputs a state |ψ ′ ⟩ and V outputs a decision bit b ∈ {Accept, Reject}. The scheme must satisfy the following properties:

• Correctness. For every message m, every key pair (pk,sk) in the support of KeyGen(1<sup>λ</sup> ), and every verification key vk in the support of VKGen(1<sup>λ</sup> ),

$$\Pr\left[m = \mathsf{Dec}(\mathsf{sk}, \mathsf{Enc}(\mathsf{pk}, m, \mathsf{vk})] = 1 - \mathsf{negl}(\lambda)\right]$$

• Semantic Security. For every pair of messages (m0, m1),

$$\left\{ (\mathsf{ct}_0, \mathsf{vk}) : \begin{array}{c} (\mathsf{pk}, \mathsf{sk}) \leftarrow \mathsf{KeyGen}(1^\lambda) \\ \mathsf{vk} \leftarrow \mathsf{VKGen}(1^\lambda) \\ \mathsf{ct}_0 \leftarrow \mathsf{Enc}(\mathsf{pk}, m_0, \mathsf{vk}) \end{array} \right\} \approx_c \left\{ (\mathsf{ct}_1, \mathsf{vk}) : \begin{array}{c} (\mathsf{pk}, \mathsf{sk}) \leftarrow \mathsf{KeyGen}(1^\lambda) \\ \mathsf{vk} \leftarrow \mathsf{VKGen}(1^\lambda) \\ \mathsf{ct}_1 \leftarrow \mathsf{Enc}(\mathsf{pk}, m_1, \mathsf{vk}) \end{array} \right\}$$

{66}------------------------------------------------

• PoNI Correctness and State Preservation: Let (pk,sk) be in the support of KeyGen(1<sup>λ</sup> ), let m be an arbitrary message, let vk be in the support of VKGen, and let |ct⟩ be a ciphertext which decrypts to m with certainty.

$$(\ket{\psi'}, b) \leftarrow \mathsf{PoNI}\langle P(\ket{\mathsf{ct}}), V(\mathsf{vk}) \rangle$$

b = Accept and |ψ⟩ belongs to the space of ciphertexts which decrypt to m.

• PoNI Security: The construction satisfies either search PoNI security (Definition [18\)](#page-66-0) or decisional PoNI security (Definition [19\)](#page-66-1).

For security, we consider a general scenario where the server receives a ciphertext, then splits it into two pieces after interacting in many PoNIs. After the split, the server simultaneously gives a PoNI using one piece while the "hacker" receives the secret key and attempts to decrypt the ciphertext using the other piece.

<span id="page-66-0"></span>Definition 18 (Proofs of No-Intrusion for Encryption: Search Security). Consider the following security game PoNI-Enc-Sn(A), played by an adversary A = (A1, A<sup>P</sup> , AH) consisting of three QPT algorithms (with auxiliary quantum input) and parameterized by a non-negative integer n.

- 1. Sample a message m ← {0, 1} λ , a key pair (pk,sk) ← KeyGen(1<sup>λ</sup> ), a verification key vk ← VKGen(1<sup>λ</sup> ), and a ciphertext ct ← Enc(pk, m, vk).
- 2. Initialize A<sup>1</sup> with (pk, ct).
- 3. Perform PoNI⟨A1, V(vk)⟩ a total of n times. If the verifier outputs Reject in any of these executions, the adversary immediately loses (output 0).
- 4. A<sup>1</sup> outputs two registers (H,P).
- 5. Perform b ← PoNI⟨A<sup>P</sup> (P), V(vk)⟩, where b is the verifier's decision bit.
- 6. Run m′ ← AH(H,sk, vk).
- 7. Output 1 (the adversary wins) if b = Accept and m′ = m. Otherwise output 0 (the adversary loses).

We say the PoNI has n-time search security if for all QPT adversaries A,

$$\Pr[1 \leftarrow \mathsf{PoNI}\text{-}\mathsf{Enc}\text{-}\mathsf{S}_n(\mathcal{A})] = \mathsf{negl}(\lambda)$$

If this holds for all n = poly(λ), then we simply say that the PoNI has search security. If this holds for QPT A<sup>1</sup> and A<sup>P</sup> , but unbounded AH, we say that the PoNI has (n-time) everlasting search security.

<span id="page-66-1"></span>Definition 19 (Proofs of No-Intrusion for Encryption: Decisional Security). Consider the following security game PoNI-Secn(m), played by an adversary A(|ψ⟩) and parameterized by a message bit m ∈ {0, 1} along with a non-negative integer n.

{67}------------------------------------------------

- 1. Sample a key pair (pk,sk) ← KeyGen(1<sup>λ</sup> ), a verification key vk ← VKGen(1<sup>λ</sup> ) and a ciphertext ct ← Enc(pk, m, vk).
- 2. Initialize A(|ψ⟩) with (pk, ct).
- 3. Perform PoNI⟨A(· · ·), V(vk)⟩ a total of n times. If the verifier outputs Reject in any of these executions, immediately output ⊥.
- 4. A outputs a register RDec.
- 5. Perform PoNI⟨A(· · ·), V(vk)⟩. If the prover rejects, output ⊥. Otherwise output (RDec,sk, vk).

We say the PoNI has n-time security if for all QPT adversaries A(|ψ⟩),

$$\{\mathsf{PoNI-Sec}_n(0)\} \approx \{\mathsf{PoNI-Sec}_n 1)\}$$

If this holds for all n = poly(λ), then we simply say that the PoNI is secure. If these two distributions are instead statistically close, we say that the PoNI is (n-time) everlasting secure.

Goyal and Raizes show that search PoNI security can be generically compiled to decision PoNI security in the quantum random oracle model (QROM).

Theorem 15. Any PKE with search PoNI security can be generically compiled into a PKE with decisional PoNI security in the quantum random oracle model.

### <span id="page-67-0"></span>A.2 Tools

PKE with Publicly Verifiable Certified Deletion. The construction uses a PKE with publicly-verifiable certified deletion, which are known from a variety of assumptions including post-quantum PKE [\[BKP23,](#page-62-8) [BGK](#page-62-9)<sup>+</sup>24, [BKM](#page-62-5)<sup>+</sup>23, [KNY23\]](#page-63-4). Roughly, this primitive allows generation of a quantum ciphertext that can be destructively measured to produce a certificate cert. The certificate can be checked by anyone using a public verification key; if the certificate is valid, then the ciphertext can no longer be decrypted, even using the secret decryption key.

Definition 20 (PKE with Publicly Verifiable Certified Deletion). A PKE scheme with publicly verifiable certified deletion has the following syntax.

- (sk, pk) ← KeyGen(1<sup>λ</sup> ) takes as input the security parameter then outputs a secret decryption key sk and public encryption key pk.
- (|ct⟩, vk) ← Enc(pk, m) takes as input a public key and a message bit b then outputs a ciphertext and an associated verification key.
- m ← Dec(sk, |ct⟩) takes as input a secret key and a ciphertext then outputs a message.

{68}------------------------------------------------

- $cert \leftarrow Del(|ct\rangle)$  takes as input a ciphertext then outputs a (classical) certificate.
- Accept/Reject  $\leftarrow$  Ver(vk, cert) takes as input a verification key and a certificate, then outputs accept or reject.

It must satisfy the standard decryption correctness and semantic security properties for PKE.

Additionally, it must satisfy correctness of deletion: for any message m,

$$\Pr\left[ \begin{array}{c} (\mathsf{sk},\mathsf{pk}) \leftarrow \mathsf{KeyGen}(1^\lambda) \\ \mathsf{Accept} \leftarrow \mathsf{Ver}(\mathsf{vk},\mathsf{cert}) : \ (|\mathsf{ct}\rangle\,,\mathsf{vk}) \leftarrow \mathsf{Enc}(\mathsf{pk},m) \\ \mathsf{cert} \leftarrow \mathsf{Dec}(\mathsf{sk},\mathsf{kct}) \end{array} \right] = 1 - \mathsf{negl}(\lambda)$$

Finally, it must also satisfy **publicly verifiable certified deletion security**: for every QPT adversary A,

$$\frac{1}{2}\|\mathsf{PVCD}\text{-}\mathsf{Exp}(0) - \mathsf{PVCD}\text{-}\mathsf{Exp}(1)\|_1 = \mathsf{negl}(\lambda)$$

where the experiment PVCD-Exp(b) is defined as follows:

- 1. Sample keys  $(\mathsf{sk}, \mathsf{pk}) \leftarrow \mathsf{KeyGen}(1^{\lambda})$  and  $encrypt(|\mathsf{ct}\rangle, \mathsf{vk}) \leftarrow \mathsf{Enc}(\mathsf{pk}, m)$ .
- 2. Run  $\mathcal{A}(|\mathsf{ct}\rangle, \mathsf{pk}, \mathsf{vk})$  and parse their output as a certificate  $\mathsf{cert}$  and a state on register  $\mathcal{R}_{\mathcal{A}}$ .
- 3. If Ver(vk, cert) = Accept, output  $\mathcal{R}_{\mathcal{A}}$ . Otherwise output  $\perp$ .

**Simultaneous Extraction.** Additionally, the analysis requires the following lemma from [GR25]. Informally, it states that if two adversaries B and C simultaneously succeed in a search task with noticeable probability without communicating, then they also simultaneously succeed with noticeable probability even when C's challenge is modified in a way which is indistinguishable to C, but which may be distinguishable to B.

<span id="page-68-0"></span>**Lemma 10.** Let  $\mathcal{D}$  be a distribution outputting three registers  $(\mathcal{A}, \mathcal{B}, \mathcal{C})$  along with a classical string  $k_C$ . Let  $\mathcal{D}'$  be a distribution taking as input  $k_C$  then outputting  $\mathcal{C}'$  such that

$$\{(\mathcal{A}, \mathcal{C}, k_C) : (\mathcal{A}, \mathcal{B}, \mathcal{C}, k_C) \leftarrow \mathcal{D}\} \approx_c \left\{ (\mathcal{A}, \mathcal{C}', k_C) : \begin{array}{c} (\mathcal{A}, \mathcal{B}, \mathcal{C}, k_C) \leftarrow \mathcal{D} \\ \mathcal{C}' \leftarrow \mathcal{D}'(k_C) \end{array} \right\}$$

For all QPT algorithms (A, C) and all (unbounded) quantum algorithms B, if

$$\Pr\left[1 \leftarrow B(\mathcal{B}, \mathcal{A}_B) \land 1 \leftarrow C(\mathcal{C}, k_C, \mathcal{A}_C) : \begin{array}{c} (\mathcal{A}, \mathcal{B}, \mathcal{C}, k_C) \leftarrow \mathcal{D} \\ (\mathcal{A}_B, \mathcal{A}_C) \leftarrow A(\mathcal{A}) \end{array}\right] \ge 1/p$$

for some  $p = poly(\lambda)$ , then there exists  $q = poly(\lambda)$  such that

$$\Pr\left[1 \leftarrow B(\mathcal{B}, \mathcal{A}_B) \land 1 \leftarrow C(\mathcal{C}', k_C, \mathcal{A}_C) : \begin{array}{c} (\mathcal{A}, \mathcal{B}, \mathcal{C}, k_C) \leftarrow \mathcal{D} \\ (\mathcal{A}, \mathcal{B}, \mathcal{C}, k_C) \leftarrow \mathcal{A}(\mathcal{A}) \\ (\mathcal{C}' \leftarrow \mathcal{D}'(k_C) \end{array}\right] \geq 1/q$$

#### <span id="page-69-0"></span>A.3 Construction

We construct a PKE with *search* PoNI security. As mentioned previously, any such scheme can be generically transformed into one with decisional PoNI security in the quantum random oracle model. We make use of our specific construction of state-preserving arguments for NP to enable compatibility with Lemma 10.<sup>29</sup> The construction uses the following tools:

- A PKE scheme with certified deletion PKE.(KeyGen, Enc, Dec, Del, Ver).
- A signature scheme Sig.(KeyGen, Sign, Ver).
- The state-preserving argument for NP SPA<sub>NP</sub> from Construction 2.

<span id="page-69-2"></span>Construction 7. The core algorithms are as follows:

- KeyGen(1 $^{\lambda}$ ) samples and outputs (sk, pk)  $\leftarrow$  PKE.KeyGen(1 $^{\lambda}$ ).
- VKGen(1 $^{\lambda}$ ) samples a signing key pair ( $\mathsf{sk}_{\mathsf{Sig}}, \mathsf{pk}_{\mathsf{Sig}}$ )  $\leftarrow \mathsf{Sig}.\mathsf{KeyGen}(1^{\lambda})$ . It outputs  $\mathsf{vk} = (\mathsf{sk}_{\mathsf{Sig}}, \mathsf{pk}_{\mathsf{Sig}})$ .
- $\mathsf{Enc}(\mathsf{pk}, m; \mathsf{vk})$  parses  $\mathsf{vk} = (\mathsf{sk}_{\mathsf{Sig}}, \mathsf{pk}_{\mathsf{Sig}})$ ). Then, it samples  $x \leftarrow \{0, 1\}^{\lambda}$  and encrypts  $(|\mathsf{ct}\rangle, \mathsf{vk}') \leftarrow \mathsf{PKE}.\mathsf{Enc}(\mathsf{pk}, x)$ . Finally, it signs  $\sigma \leftarrow \mathsf{Sig}.\mathsf{Sign}(\mathsf{sk}_{\mathsf{Sig}}, \mathsf{vk}')$  and outputs

$$|\mathsf{ct}'\rangle = (|\mathsf{ct}\rangle\,,\;\mathsf{vk}',\;\sigma)$$

•  $\mathsf{Dec}(\mathsf{sk}, |\mathsf{ct'}\rangle)$  parses  $|\mathsf{ct'}\rangle = (|\mathsf{ct}\rangle, \mathsf{vk'}, \sigma)$ , then computes  $m \leftarrow \mathsf{PKE}.\mathsf{Dec}(\mathsf{sk}, |\mathsf{ct}\rangle)$ . It outputs m.

The **proof of no-intrusion** is as follows.

- 1. **Prover:** Parse  $|\mathsf{ct'}\rangle = (|\mathsf{ct}\rangle, \mathsf{vk'}, \sigma)$  and send  $(\mathsf{vk'}, \sigma)$  to the verifier. Coherently compute  $\mathsf{Del}(|\mathsf{ct}\rangle)$  to obtain  $\sum_{\mathsf{cert}} \alpha_{\mathsf{cert}} |\mathsf{cert}, \mathsf{garbage}_{\mathsf{cert}}\rangle$ .
- 2. **Verifier:** Parse  $vk = (sk_{Sig}, pk_{Sig})$ . Then, compute  $Sig.Ver(pk_{Sig}, \sigma, vk')$ . If it rejects, immediately output reject.
- 3. Both. Interact in an execution of SPA<sub>NP</sub> for the language

**Theorem 16.** Construction 7 is a PKE with everlasting search proofs of no-intrusion, assuming the existence of PKE with publicly verifiable certified deletion (which can be based on post-quantum PKE [BKM<sup>+</sup>23, KNY23]), post-quantum signature schemes, and that Construction 2 is a state-preserving argument of knowledge for NP (all of which can be based on LWE).

To see PoNI security, first observe that the adversary may simulate any number of PoNIs locally using  $vk_{Sig}$ . Furthermore, in the final PoNI, the NP statement proved in the  $SPA_{NP}$  execution must be for the original vk' generated at encryption time, or else the unforgeability of Sig is broken.

We show PoNI-Enc-S search security by reducing to the publicly verifiable certified deletion property of PKE. Specifically, given a PoNI-Enc-S search adversary  $(A_1, A_P, A_H)$ , the reduction plays the certified deletion security game as follows.

- 1. Sample random  $m_0 \leftarrow \{0,1\}^{\lambda}$  and  $m_1 \leftarrow \{0,1\}^{\lambda}$  to be the challenge plaintexts. Receive  $(|ct_b\rangle, vk')$ , an encryption of a random  $m_b$  along with the corresponding verification key, from the certified deletion challenger.
- 2. Sample  $vk = (sk_{Sig}, pk_{Sig})$  and compute  $\sigma = Sig.Sign(sk_{Sig}, vk')$  to complete the ciphertext  $|ct'\rangle = (|ct_b\rangle, vk', \sigma)$ , then give it to  $A_1$ .
- 3. Run the initial PoNI executions locally with  $A_1$ .
- 4. Receive the adversary's split registers  $(\mathcal{H}, \mathcal{P})$ .
- <span id="page-70-0"></span>5. Execute the final PoNI with  $\mathcal{A}_P(\mathcal{P})$  using the argument of knowledge extractor  $\mathsf{Ext}(\mathcal{A})$  for  $\mathsf{SPA}_{\mathsf{NP}}$ . If it succeeds in extraction, it outputs  $\mathsf{cert}$  such that  $\mathsf{PKE}.\mathsf{Ver}(\mathsf{vk'},\mathsf{cert}) = \mathsf{Accept}$  (since  $\mathsf{vk'}$  is the statement used as argued previously). If extraction fails, set  $\mathsf{cert} = \bot$ .
- 6. Send cert and  $\mathcal{H}$  to the certified deletion challenger.
- <span id="page-70-1"></span>7. Receive either  $\mathcal{H}$  or  $\bot$  from the certified deletion challenger, depending on whether cert is valid.
- 8. If the latter, output a random bit. If the former, run  $m' \leftarrow \mathcal{A}_H(\mathcal{H})$ . Then, output b if  $m' = m_b$  and otherwise guess randomly.

We now analyze the reduction's advantage in the certified deletion security game, supposing that the adversary violates PoNI-Enc-S search security for the sake of contradiction. Observe that  $\mathcal{A}_H$ 's view is independent of  $m_{1-b}$ , so  $m' = m_{1-b}$  with negligible probability.

Since the reduction guesses randomly in this case unless m′ = m<sup>b</sup> or if cert is invalid, it suffices to show that with noticeable probability m′ = m<sup>b</sup> and simultaneously cert is valid.

Consider the hybrid experiment where steps [5](#page-70-0) and [7](#page-70-1) are replaced by running the honest execution of SPANP (i.e. the honest PoNI), and running m′ ← AH(H) if the PoNI accepts (instead of checking an extracted cert). This matches the PoNI-Enc-S game for random message mb, so by assumption A<sup>H</sup> outputs m′ = m<sup>b</sup> with noticeable probability.[30](#page-71-0)

Now consider the following simultaneous search task: A<sup>1</sup> acts as the splitter A; AH(H) acts as (unbounded) B and searches for mb; C takes as input a view from A<sup>P</sup> just before the opening phase in Construction [2](#page-37-1) and searches for an opening that would lead to an accepting response. This simultaneous search task matches the hybrid experiment described above, so B and C succeed with noticeable probability. Now consider replacing C's input by a view from A<sup>P</sup> in an execution of SPANP where the public parameters are generated in injective mode (this matches the extractor from the proof of Theorem [10\)](#page-38-0). This view is computationally indistinguishable to C because the trapdoor has not yet been revealed in the SPANP execution. Therefore Lemma [10](#page-68-0) implies that B and C simultaneously succeed in their search tasks with noticeable probability even when C receives the modified view. Finally, we argue that if C succeeds in finding an accepting opening, then with overwhelming probability Ext successfully extracts a witness. This follows from the fact that if Ext would fail, then for the overwhelming majority of challenges in SPANP, there does not exist an accepting opening.

Therefore with noticeable probability, Ext extracts a valid cert and A<sup>H</sup> outputs m′ = mb. As argued previously, this contradicts the publicly verifiable certified deletion security of PKE.
