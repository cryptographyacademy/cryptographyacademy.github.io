# A Trace Based *GF*(2*<sup>n</sup>* ) Inversion Algorithm

Haining Fan fhn@tsinghua.edu.cn

### Abstract

By associating Fermat's Little Theorem based *GF*(2*<sup>n</sup>* ) inversion algorithms with the multiplicative Norm function, we present an additive Trace based *GF*(2*<sup>n</sup>* ) inversion algorithm. For elements with Trace value 0, it needs 1 less multiplication operation than Fermat's Little Theorem based algorithms in some *GF*(2*<sup>n</sup>* )s.

### Index Terms

Finite field, Inversion algorithm, Norm, Trace.

Efficient implementation of *GF*(2*n*) inversion is important for practical applications and can be found in, for example, Feng's algorithm [1] (received on March 13, 1987 and published in October 1989), which "requires the same number of multiplications as Itoh and Tsujii's algorithm" [2] in [3] (received on July 8, 1987 and published in 1988). These algorithms are based on the fact that *GF*(2*n*) *âˆ—* is a cyclic group of order 2 *<sup>n</sup> âˆ’* 1, i.e., *âˆ€A âˆˆ GF*(2*n*) *âˆ—* ,

*âŒŠ*log<sup>2</sup> (*n âˆ’* 1)*âŒ‹* + *HammingW eight*(*n âˆ’* 1) *âˆ’* 1 multiplications and *n âˆ’* 1 squarings.

The above *Aâˆ’*<sup>1</sup> expression itself is close to that of the multiplicative Norm function, which is defined as

$$Norm(A) = \prod_{i=0}^{n-1} A^{2^i}.$$

This viewpoint leads us to consider the additive absolute Trace function, which is defined as

$$Tr(A) = \sum_{i=0}^{n-1} A^{2^i}.$$

We now give some examples to show the computational produce of this formula for *A* such that *T r*(*A*) = 0.

#### A. Example *GF*(2<sup>3</sup> )

Because 0 = *T r*(*A*) = *A* + *A*<sup>2</sup> + *A*<sup>4</sup> , we have *A* = *A*<sup>2</sup> + *A*<sup>4</sup> and *Aâˆ’*<sup>1</sup> = 1+*A*<sup>2</sup> .

We note that the above "1+ " operation in a polynomial basis is only a bit NOT operation, and can be merged into a VLSI squarer.
