# CRISP: Circuit-pRivate Single-Image Steganography with Permutations

Shahzad Ahmad [0000−0002−9654−869X] and Stefan Rass [0000−0003−2821−2489]

LIT Secure and Correct Systems Lab, Johannes Kepler University Linz, Altenbergerstrasse 69, 4040 Linz, Austria shahzad.ahmad@jku.at, stefan.rass@jku.at

Abstract. We introduce CRISP (Circuit-pRivate Single-Image Steganog raphy with Permutations), a provably secure homomorphic steganography scheme designed to address key limitations in existing approaches to private computation. Current methods often suffer from excessive image overhead and a complete lack of circuit privacy. Specifically, many techniques require a separate cover image for each wire in a circuit, leading to a large number of images for even moderately sized circuits. This also inadvertently reveals the underlying logical gate structure to an honestbut-curious cloud provider, as wire roles are often fixed to specific images. CRISP resolves these challenges by embedding all three Fredkin gate inputs directly into the RGB channels of a single cover image at a secret pixel position (row, col). Similarly, all three outputs are written to a single output image. To ensure robust circuit privacy, CRISP employs two independently sampled permutations: πin assigns input channel roles, and πout reassigns output channel roles. These permutations work in tandem to prevent an adversary from deducing the circuit's structure. Without πout varying per gate, the Fredkin gate's pass-through of the control bit could enable a deterministic channel-matching attack, fully exposing the logical gate structure to the cloud provider. By resampling πin and πout independently for each gate, CRISP effectively prevents cross-gate correlation attacks.

Our security analysis demonstrates that an adversary's advantage in locating the secret pixel and inferring the embedded bits decays as Θ(1/(h×w)), where h×w is the image resolution. This bound is cryptographically negligible for any practical image size. CRISP significantly reduces image overhead: per gate, it uses just two images (one input cover plus one independent output cover). This represents a substantial reduction compared to other methods that might require multiple images per wire. For an obfuscated benchmark circuit, CRISP further demonstrates a notable reduction in total image count. Crucially, CRISP delivers circuit privacy, a qualitatively new and essential security guarantee.

The correctness, security, and efficiency of CRISP are formally proved and experimentally validated.

Keywords: Homomorphic steganography · Circuit privacy · Fredkin gate · privacypreserving computation · Cloud computing · Reversible logic

## <span id="page-1-0"></span>1 Introduction

Cloud computing has transformed how organisations store and process sensitive data. Outsourcing computation to third-party cloud providers offers scalability and cost efficiency, but introduces a significant privacy challenge: enabling an untrusted server to compute on data without revealing any information about it.

Existing solutions address this from different angles. On the one hand, Fully Homomorphic Encryption (FHE) [\[18\]](#page-20-0) enables arbitrary computation over encrypted data, but its computational overhead remains prohibitive for large-scale deployment. On the other hand, Trusted Execution Environments (TEEs) [\[12\]](#page-20-1) offer hardware-level isolation but depend on specific CPU features, require complex attestation protocols, and remain vulnerable to side-channel attacks [\[6\]](#page-20-2). Meanwhile, traditional steganography hides the existence of a message but provides no mechanism for computation on hidden data.

Provably Secure Homomorphic Steganography (ProSt) [\[2\]](#page-19-0) introduces a new approach: hiding secret bits in cover images via LSB steganography and enabling homomorphic computation on them using Fredkin gates [\[16\]](#page-20-3). An honest-butcurious cloud provider processes each pixel without knowing which carries the secret. ProSt offers provable security; the adversary's advantage decreases as candidate pixel-channel positions increase.

ProSt faces two key limitations. First, each circuit wire needs a dedicated cover image. A circuit with 61 wires requires 61 images, creating significant communication and storage overhead. Second, ProSt lacks circuit privacy: since wire roles are tied to specific images, the cloud can always identify which image holds the control bit at each gate, exposing the computation's logical structure. Our Contributions. This paper introduces CRISP, which resolves both limitations through:

- 1. Single-image Fredkin embedding. All three Fredkin gate inputs are embedded into the three RGB channels of a single input cover image at a secret pixel position (row, col). All three outputs are written into a single, independently sampled output cover image at the same position. Each gate consumes exactly 2 images, one input cover and one output cover, compared to ProSt's 6 per gate (three input wire images and three output wire images). For an n-gate circuit, CRISP requires 2n images in total, versus ProSt's L images for L circuit wires, giving a 3× per-gate reduction and a L/(2n) circuit-level reduction.
- 2. Dual independent permutations. A secret input permutation πin ∈ S<sup>3</sup> assigns the three logical roles (control, x, y) to physical channels {R, G, B} before embedding. A separate, independently sampled output permutation πout ∈ S<sup>3</sup> reassigns channel roles when the cloud writes Fredkin outputs. Both permutations are part of the public circuit specification: Carol needs πin to read the correct logical inputs and πout to write the correct outputs. The circuit privacy guarantees arise from πout varying independently per gate.

{2}------------------------------------------------

- 3. Formal security analysis with Bayesian advantage decay. We prove that the adversary's advantage in inferring the embedded secret is Θ(1/n) with n = h×w. The proof is self-contained and relies on a direct Bayesian calculation that exploits the exchangeability of the image LSB vector.
- 4. Necessity of dual permutation for circuit privacy. We formally show that a single permutation is insufficient: the Fredkin gate's pass-through of the control bit enables a channel-matching attack that deterministically identifies the control channel without πout.

## <span id="page-2-0"></span>2 Related Work

Steganography. LSB substitution [\[9\]](#page-20-4) replaces the least significant bit of pixel values with secret bits, achieving high capacity with negligible perceptual distortion. More sophisticated spatial-domain techniques including pixel-value differencing [\[29\]](#page-21-0) and adaptive methods [\[26\]](#page-21-1) improve resistance to steganalysis. Transform-domain approaches such as DCT-based JPEG steganography and DWT-based schemes hide data in frequency coefficients for stronger statistical imperceptibility. Deep learning methods [\[3,](#page-19-1)[32\]](#page-21-2) jointly train encoder-decoder networks for end-to-end imperceptibility and robustness. All of these methods treat steganography purely as a hiding mechanism: none support computation on hidden data without first extracting the secret.

Homomorphic Encryption. Gentry's seminal work [\[18\]](#page-20-0) established FHE, enabling the arbitrary evaluation of circuits over ciphertexts. Subsequent schemes, including BGV [\[5\]](#page-19-2), BFV [\[15\]](#page-20-5), CKKS [\[10\]](#page-20-6), and TFHE [\[11\]](#page-20-7), significantly improved practical efficiency. Despite these advances, FHE incurs substantial computational overhead due to bootstrapping, complex key management, and ciphertext expansion. CRISP avoids encryption entirely, instead exploiting LSB statistical properties to achieve privacy during computation.

{3}------------------------------------------------

Trusted Execution Environments. Intel SGX [12], ARM TrustZone, and AMD SEV provide hardware-enforced isolation for sensitive computations. However, they depend on specific hardware features, require complex remote attestation, and remain vulnerable to side channel attacks [6,25]. CRISP requires no specialised hardware and is immune to these concerns since the cloud provider computes openly on all pixels.

Secure Multi-Party Computation. Yao's garbled circuits [30] and the GMW protocol [20] established MPC foundations, with subsequent work including SPDZ [13], improving efficiency. MPC protocols generally require multiple rounds of interaction, which is impractical for asynchronous cloud outsourcing. CRSIP requires zero interaction between Alice and the cloud during computation.

Garbled Circuits and Circuit Privacy. Circuit privacy in garbled circuits [30,4] ensures that a garbled circuit reveals nothing about the circuit topology beyond its input-output behaviour. CRISP introduces an analogous notion in the steganographic setting: the cloud provider cannot determine which channel plays which logical role at any gate. This is achieved without encryption overhead.

*ProSt.* ProSt introduced [2] homomorphic steganography, enabling provably secure computation on LSB-hidden data using Fredkin gates. ProSt's per-wire per-image architecture and lack of circuit privacy are the direct motivations for CRISP.

## <span id="page-3-0"></span>3 Background

#### 3.1 LSB Steganography

An image I is a three-dimensional tensor  $I \in \mathbb{Z}_{256}^{h \times w \times 3}$ , where h, w are height and width in pixels and the third dimension indexes channels  $\{0, 1, 2\} = \{R, G, B\}$ .

**Definition 1 (LSB Embedding and Extraction).** For image I, secret bit  $b \in \{0,1\}$ , and position (row, col, k):

$$\operatorname{Emb}(I, b, row, col, k) : I[row, col, k] \leftarrow (I[row, col, k] \& \neg 1) \mid b \tag{1}$$

$$\operatorname{Ext}(I, row, col, k) : b \leftarrow I[row, col, k] \& 1$$
 (2)

The modification changes pixel intensity by at most 1, which is imperceptible. Security relies not on imperceptibility alone but on positional secrecy: without knowing (row, col), an adversary cannot identify which of the  $h \times w$  pixels carries the secret.

#### 3.2 The Fredkin Gate

The Fredkin gate  $F: \{0,1\}^3 \to \{0,1\}^3$  is a universal reversible logic gate defined as:

$$F(c, x, y) = \begin{cases} (c, x, y) & \text{if } c = 0\\ (c, y, x) & \text{if } c = 1 \end{cases}$$
 (3)

{4}------------------------------------------------

where c is the control bit and x, y are data bits. Two properties are central to CRISP.

Lemma 1 (Universality [\[16\]](#page-20-3)). The Fredkin gate can simulate AND, OR, and NOT:

$$AND(a, b) = F(a, b, 0)_y$$
  $NOT(x) = F(x, 1, 0)_x$   $OR(a, b) = F(a, 1, b)_y$ 

where subscripts denote output wire selection. Any Boolean function can be expressed as a network of Fredkin gates.

<span id="page-4-1"></span>Lemma 2 (Hamming Weight Invariance [\[2\]](#page-19-0)). For any (c, x, y) ∈ {0, 1} 3 , the Hamming weight satisfies c+x+y = cout+xout+yout, where (cout, xout, yout) = F(c, x, y).

#### 3.3 ProSt Summary

ProSt [\[2\]](#page-19-0) assigns one cover image per circuit wire. For each gate G<sup>i</sup> with inputs (Ic, I1, I2) and outputs (Oc, O1, O2), the cloud applies:

$$(\operatorname{lsb}(O_c[r,c,k]), \operatorname{lsb}(O_1[r,c,k]), \operatorname{lsb}(O_2[r,c,k])) \leftarrow F(\operatorname{lsb}(I_c[r,c,k]), \\ \operatorname{lsb}(I_1[r,c,k]), \operatorname{lsb}(I_2[r,c,k]))$$

at every pixel (r, c) and channel k. Alice embeds her secret into each image at an independently sampled position (r<sup>i</sup> , c<sup>i</sup> , ki) from h × w × 3 possibilities. Limitations. (i) ProSt allocates one cover image per circuit wire. For each Fredkin gate, the cloud accesses 3 input wire images and writes to 3 output wire images, for a total of 6 images per gate. Input wire images are shared across gates (e.g., ancillary wires such as const<sup>0</sup> and const<sup>1</sup> appear as inputs to many gates), so the total distinct image count equals the wire count L, not 6n. For the 17-gate obfuscated benchmark, L = 61. (ii) Wire roles are fixed to specific images, so the cloud always knows which image carries the control bit at each gate; there is no circuit privacy. CRISP addresses both.

## <span id="page-4-0"></span>4 CRISP: Formal Scheme Definition

### 4.1 Overview

CRISP maps the three inputs (c, x, y) of a Fredkin gate to the three RGB channels of a single input image Iin at a secret pixel position (row, col), and maps the three outputs to the three channels of a single output image Iout at the same pixel position. Two independently sampled permutations control channel-to-role assignment: πin ∈ S<sup>3</sup> assigns logical roles to channels before embedding, and πout ∈ S<sup>3</sup> assigns output roles after Fredkin computation. Both permutations are part of the public circuit specification transmitted to Carol; Carol requires πin ∈ S<sup>3</sup> to read the correct logical inputs and πout ∈ S<sup>3</sup> to write outputs to the correct channels. The secret known only to Alice and Bob is the pixel position (row, col).

### 4.2 System Parameters

Let images have fixed dimensions h×w ×3. A Boolean circuit C is a DAG with:

- Input wires Win, intermediate wires Wmid, output wires Wout
- n Fredkin gates G = {G1, . . . , Gn} in valid topological order
- Each gate G<sup>i</sup> has input triple (c<sup>i</sup> , x<sup>i</sup> , yi) and output triple (c out i , x out i , y out i )

### 4.3 Secret Key

The secret key shared between Alice and Bob is:

$$K = \left\{ (row_i, col_i) \right\}_{i=1}^n \tag{4}$$

where (row<sup>i</sup> , col <sup>i</sup>) ∈ {0, . . . , h−1} × {0, . . . , w−1} is the secret pixel position for gate G<sup>i</sup> . The permutations π (i) in , π (i) out ∈ S<sup>3</sup> are independently sampled per gate and included in the public circuit specification. K is communicated from Alice to Bob over a private channel before any computation.

## 4.5 Remark on Carol's Knowledge of $\pi_{in}$ and $\pi_{out}$

Both  $\pi_{\rm in}$  and  $\pi_{\rm out}$  are included in the circuit specification transmitted to Carol. Under the honest-but-curious model, Carol follows the protocol exactly. She requires  $\pi_{\rm in}$  to read the correct logical roles from the input channels (Algorithm 4, lines 2–4), and  $\pi_{\rm out}$  to write each output bit to the correct channels (lines 8–10). Without the secret pixel position (row, col), Carol cannot identify which pixel carries the actual secret, even with full knowledge of both permutations. The sole secret in CRISP is the position (row, col), which is never transmitted and is known only to Alice and Bob.

The circuit privacy guarantees do not stem from hiding  $\pi_{\rm in}$ ; rather, they arise from the fact that  $\pi_{\rm out}$  is resampled independently per gate, which breaks the per-channel LSB correlation attack described in Theorem 2 (see also Remark 1 for the precise scope of this guarantee).

#### 4.6 Multi-Gate Circuit Evaluation

For a circuit with n gates in topological order:

- 1. Key Generation: Alice runs LocGen for each gate  $G_i$ , producing  $K = \{(row_i, col_i)\}_{i=1}^n$ . She also runs PermGen per gate to generate  $(\pi_{in}^{(i)}, \pi_{out}^{(i)})$ , which are included in the public circuit specification.
- 2. Embedding: Alice embeds the circuit input bits into  $I_{\text{in}}^{(1)}$  at position  $(row_1, col_1)$  using  $\pi_{\text{in}}^{(1)}$ . Subsequent gate inputs propagate automatically through Carol's pixel-wise computation; no re-embedding is required.
- 3. Computation: Carol evaluates gates in topological order. For gate  $G_i$ , she applies  $\text{Comp}(I_{\text{in}}^{(i)}, I_{\text{out}}^{(i)}, \pi_{\text{in}}^{(i)}, \pi_{\text{out}}^{(i)})$ . The output image  $I_{\text{out}}^{(i)}$  serves as  $I_{\text{in}}^{(i+1)}$  for linear chains. For general DAGs, wire tracking routes outputs to the correct subsequent gate inputs.
- 4. Extraction. Bob applies Ext to  $I_{\text{out}}^{(n)}$  at position  $(row_n, col_n)$  with  $\pi_{\text{out}}^{(n)}$  to recover the final output bits.

### 4.7 Image Count

ProSt allocates one cover image per circuit wire and maps each wire name to its own image throughout computation. For a Fredkin gate with inputs (c-wire, x-wire, y-wire) and outputs (cout-wire, xout-wire, yout-wire), ProSt reads three input wire images and writes three output wire images, for a total of 6 images per gate. Because ancillary wires such as const<sup>0</sup> and const<sup>1</sup> appear as inputs to many gates, these input images are shared across the circuit. The total distinct image count, therefore, equals the number of distinct wire L, not 6n.

CRISP replaces the wire-centric model with a gate-centric one. Each gate Gi is assigned one fresh input cover I (i) in and one independent output cover I (i) out : 2 images per gate. No image is shared between gates. Wire values propagate between gates as scalars in the signal dictionary, not as images.

Proposition 1 (Image Count). A circuit of n Fredkin gates requires exactly 2n cover images under CRISP (n input covers and n output covers, all independently sampled) and L images under ProSt (L = number of distinct circuit wires).

This yields two distinct and complementary reduction figures:

- Per-gate: 6/2 = 3× reduction, independent of circuit structure.
- Circuit-level: L/(2n) reduction. For the 17-gate obfuscated benchmark (n = 17, L = 61): 61/34 ≈ 1.8×.

The circuit-level figure is lower than 3× because ProSt's wire-sharing reduces its effective total, whereas CRISP's 2n images are all distinct by design. Both figures are reported in Table [6.](#page-17-0)

#### 4.8 Correctness

<span id="page-7-0"></span>Theorem 1 (Correctness). For any circuit C, any secret input bits, and any key K, the output recovered by Ext equals the correct evaluation of C on the input bits.

Proof. Consider gate G<sup>i</sup> . Alice embeds bits (c, x, y) directly (without any padding or transformation) into I (i) in at (row<sup>i</sup> , col <sup>i</sup>) using π (i) in :

$$I_{\text{in}}^{(i)}[row_i, col_i, \pi_{\text{in}}^{(i)}(t)] \text{ carries } bits[t] \text{ for } t \in \{0, 1, 2\}.$$

Carol applies Comp uniformly across every pixel of the entire h×w image. Specifically, she reads channel π (i) in (t) from every pixel position as logical input role t, applies F to the three resulting LSBs, and writes output role t into channel π (i) out (t) of the output cover at every position. Carol does not know (row<sup>i</sup> , col <sup>i</sup>) and applies no special treatment to any pixel.

Correctness at the secret pixel follows from this uniformity: because Alice wrote bits[t] into channel π (i) in (t) at (row<sup>i</sup> , col <sup>i</sup>), Carol's uniform channel-read

recovers (c, x, y) correctly at that position as a consequence of the same operation applied to all  $h \times w$  pixels. The Fredkin computation  $F(c, x, y) = (c_{\text{out}}, x_{\text{out}}, y_{\text{out}})$  is therefore correct at  $(row_i, col_i)$ , and the outputs are written to channels  $\pi_{\text{out}}^{(i)}(t)$  at that position. Bob reads channel  $\pi_{\text{out}}^{(i)}(t)$  at  $(row_i, col_i)$  to recover output t. By induction over gates in topological order, correctness propagates through the entire circuit.

## <span id="page-8-0"></span>5 Security Analysis

### 5.1 Security Model

We analyse CRISP against an honest-but-curious adversary Carol who receives all stego images plus the full circuit description including  $\pi_{\text{in}}^{(i)}$  and  $\pi_{\text{out}}^{(i)}$  for every gate (both are required for correct evaluation), but has no knowledge of the secret pixel positions  $\{(row_i, col_i)\}_{i=1}^n$ .

## Definition 2 (CRISP Security Game Game<sub>CRISP</sub>).

- 1. Setup. The challenger fixes image dimensions h, w and samples a fresh key K (Section 4).
- 2. Embedding. The challenger chooses a secret triple  $T=(c,x,y)\in\{0,1\}^3$ , runs Emb, and produces the stego image I'.
- 3. Challenge. Carol receives I', the circuit description (including  $\pi_{in}, \pi_{out}$ ), and outputs a position guess  $(row^*, col^*)$  and a value guess  $T^* \in \{0, 1\}^3$ .
- 4. Outcome. Carol wins if  $T^* = T$  and  $(row^*, col^*) = (row, col)$ .

The advantage is  $Adv_A = |Pr(T^* = T \mid M = m) - Pr(T^* = T)|$ , where M denotes the observed image LSBs and m is an arbitrary realisation.

#### 5.2 Circuit Privacy: Necessity of Per-Gate $\pi_{out}$ Resampling

We establish the precise sense in which CRISP achieves circuit privacy and what that guarantee actually means given that both  $\pi_{\rm in}$  and  $\pi_{\rm out}$  are public. Since Carol knows  $\pi_{\rm in}$ , she trivially knows which input channel holds the control bit at every gate. What CRISP's circuit privacy prevents is a different and operationally distinct threat: the ability to *statistically confirm and track* the control wire across gates by correlating LSB patterns between input and output images.

<span id="page-8-2"></span><span id="page-8-1"></span>**Definition 3 (Per-Gate Output Permutation).** CRISP uses per-gate resampling: for each gate  $G_i$ , the output permutation  $\pi_{\text{out}}^{(i)}$  is drawn uniformly and independently from  $S_3$ , independently of  $\pi_{\text{in}}^{(i)}$  and of all permutations at other gates.

With per-gate resampling, the control output is placed in channel  $\pi_{\text{out}}^{(i)}(0) \in \{0,1,2\}$  independently per gate. Aggregating  $\Delta_k$  across all pixels for gate  $G_i$  now yields  $S_k^{(i)} = 0$  for a different channel at each gate, chosen uniformly at random. Since  $\pi_{\text{out}}^{(i)}(0)$  is uniform over  $\{0,1,2\}$  and independent across gates, no cross-gate correlation of the control wire is possible through this channel-matching approach.

CRISP's circuit privacy is a precisely scoped operational property: it prevents the specific per-channel LSB-correlation attack described in Theorem 2. It does not assert simulation-based privacy in the sense of garbled circuits; Carol knows  $\pi_{\rm in}^{(i)}$  and therefore knows

which input channel carries the control bit at every gate. The guarantee is that she cannot *statistically track* the control wire across the circuit by correlating LSB patterns between successive input and output images.

#### 5.3 Information Hiding: Bayesian Advantage Bound

The main security result establishes that the adversary's advantage in inferring the embedded secret decays as  $\Theta(1/n)$  with  $n = h \times w$ . The proof is self-contained and rests on a direct Bayesian calculation.

<span id="page-11-0"></span>**Theorem 3 (Information Hiding for CRISP).** Let  $p_0 \in (0,1)$ , and let  $T = (c, x, y) \sim \text{Ber}(p_0)^3$  be a secret bit-triple. CRISP embeds T at a uniformly random pixel  $(row^*, col^*)$  chosen from  $n = h \times w$  positions; all remaining pixel LSB triples are drawn i.i.d. from  $\text{Ber}(p_0)^3$ , independently of T. Then

$$\operatorname{Adv}_{\mathcal{A}} = \frac{1 - p_t}{n} \in \Theta\left(\frac{1}{n}\right), \quad n = h \times w,$$

where  $p_t = \Pr(T = t) = p_0^{w(t)} (1 - p_0)^{3-w(t)}$  for triple t with Hamming weight w(t), and  $(1 - p_t) \in (0, 1)$  is a positive constant.

*Proof.* We proceed by a direct Bayesian calculation.

**Setup.** Represent the image LSBs as a vector  $\mathbf{M} = (M_0, M_1, \dots, M_{n-1})$  of pixel triples  $M_j \in \{0, 1\}^3$ . Since Carol knows  $\pi_{\text{in}}$ , she can recover the logical ordering of each pixel's three bits directly. The secret triple T is placed at a uniformly random position  $j^*$  drawn independently and uniformly from  $\{0, \dots, n-1\}$ , so  $\Pr(j^* = j) = 1/n$  for all j. All positions  $j \neq j^*$  hold i.i.d. decoy triples from  $\text{Ber}(p_0)^3$ , independent of T. Since T also follows  $\text{Ber}(p_0)^3$ , the vector  $\mathbf{M}$  is exchangeable: every permutation of its coordinates has the same joint distribution.

Adversary's optimal strategy. Without loss of generality, assume Carol inspects position 0 (by exchangeability, no position is distinguishable a priori). Her optimal strategy for the value guess is to output  $T^* = M_0$  (i.e. guess that position 0 holds the secret and its value equals the observed triple). We compute her advantage under this strategy.

Marginal distribution of  $M_0$ . For any triple  $t \in \{0, 1\}^3$ :

$$\Pr(M_0 = t) = \Pr(j^* = 0) \cdot \Pr(T = t) + \Pr(j^* \neq 0) \cdot \Pr(M_0 = t \mid j^* \neq 0).$$

When  $j^* = 0$ :  $M_0 = T$ , so  $\Pr(M_0 = t \mid j^* = 0) = \Pr(T = t) = p_t$ . When  $j^* \neq 0$ :  $M_0$  is an independent decoy, so  $\Pr(M_0 = t \mid j^* \neq 0) = p_t$ . Therefore:

$$\Pr(M_0 = t) = \frac{1}{n} \cdot p_t + \frac{n-1}{n} \cdot p_t = p_t.$$

**Joint probability**  $Pr(T = t, M_0 = t)$ . We condition on whether the secret is at position 0 or not:

$$\Pr(T = t, M_0 = t) = \Pr(j^* = 0) \cdot \Pr(T = t, M_0 = t \mid j^* = 0) + \Pr(j^* \neq 0) \cdot \Pr(T = t, M_0 = t \mid j^* \neq 0).$$

{12}------------------------------------------------

Case 1  $(j^* = 0)$ :  $M_0 = T$ , so  $\Pr(T = t, M_0 = t \mid j^* = 0) = \Pr(T = t) = p_t$ . Case 2  $(j^* \neq 0)$ : T and  $M_0$  are independent, so  $\Pr(T = t, M_0 = t \mid j^* \neq 0) = p_t^2$ .

Combining:

$$\Pr(T = t, M_0 = t) = \frac{1}{n} \cdot p_t + \frac{n-1}{n} \cdot p_t^2 = p_t \left( \frac{1}{n} + \frac{n-1}{n} \cdot p_t \right).$$

Posterior probability. Applying Bayes' theorem:

$$\Pr(T = t \mid M_0 = t) = \frac{\Pr(T = t, M_0 = t)}{\Pr(M_0 = t)} = \frac{p_t \left(\frac{1}{n} + \frac{n-1}{n} \cdot p_t\right)}{p_t} = \frac{1}{n} + \frac{n-1}{n} \cdot p_t.$$

Rearranging:

$$\Pr(T = t \mid M_0 = t) = p_t + \frac{1 - p_t}{n}.$$

**Advantage.** The prior probability of guessing the correct value is  $Pr(T^* = T) = p_t$ . Therefore:

$$Adv_{\mathcal{A}} = Pr(T = t \mid M_0 = t) - p_t = \frac{1 - p_t}{n}.$$

Since  $p_t \in (0,1)$  for any  $p_0 \in (0,1)$ , the factor  $(1-p_t)$  is a positive constant independent of n. Hence  $Adv_A \in \Theta(1/n)$ .

Comparison with ProSt. ProSt embeds one scalar bit at one of  $n_P = h \times w \times 3$  pixel-channel positions, giving  $Adv \in \Theta(1/n_P)$ . CRISP embeds a triple at one of  $n_C = h \times w$  pixel positions, giving  $Adv \in \Theta(1/n_C)$ . Both bounds are cryptographically negligible for any practical image. For  $128 \times 128$  images,  $n_P = 49152$  and  $n_C = 16384$ ; both  $1/n_P \approx 2.0 \times 10^{-5}$  and  $1/n_C \approx 6.1 \times 10^{-5}$  are negligible. ProSt's position space is  $3 \times$  larger and its individual bound is tighter, but CRISP compensates with a  $1.8 \times$  reduction in image count and provides the circuit privacy property of Theorem 2, absent from ProSt.

#### 5.4 Hamming Weight Preservation

<span id="page-12-0"></span>Lemma 3 (Distribution Preservation). The pixel-level LSB Hamming-weight distribution is unchanged by Comp. for every pixel (row', col'),

$$\sum_{k=0}^{2} lsb(I_{out}[row', col', k]) = \sum_{k=0}^{2} lsb(I_{in}[row', col', k]).$$

*Proof.* Lemma 2 states that F preserves Hamming weight for every input triple. Comp applies F pixel-wise, reading inputs in  $\pi_{\text{in}}$  order and writing outputs in  $\pi_{\text{out}}$  order; both are permutations of  $\{0,1,2\}$  that leave the per-pixel Hamming weight unchanged.

### 5.5 Security Under Computation

<span id="page-13-0"></span>Corollary 1 (Security Preserved Across the Circuit). Let a Boolean circuit C of depth N be composed entirely of Fredkin gates. If the inputs of C are embedded steganographically as described in Theorem [3,](#page-11-0) then the hiding property is preserved across every gate evaluation. The adversary's advantage remains in Θ(1/n) with n = h×w throughout the entire computation.

Proof. We proceed by induction on the number of gates evaluated.

Base case (N = 0): Theorem [3](#page-11-0) applies directly to the embedded inputs.

Inductive step: Assume the LSB distribution of images carrying intermediate results is identical to the initial distribution after evaluating N−1 gates. By Lemma [3,](#page-12-0) gate N preserves the per-pixel Hamming-weight distribution, so the i.i.d. Ber(p0) <sup>3</sup> property of every pixel's LSB triple is maintained. The exchangeability condition of Theorem [3](#page-11-0) therefore holds for the output image, and the advantage bound Θ(1/n) is preserved.

The advantage does not accumulate across gates. In a single execution, all n gates share the same secret position (row, col); however, each gate uses an independently sampled input cover I (i) in , output cover I (i) out , input permutation π (i) in , and output permutation π (i) out . Because the covers are independent carriers and the permutations are resampled independently per gate, the 2n image observations across the circuit are conditionally independent given the secret position. Carol's posterior over positions therefore remains uniform after all 2n observations, and the advantage stays in Θ(1/n) throughout the computation. ⊓⊔

### 5.6 Security Against Adaptive Attacks

Theorem [3](#page-11-0) and Corollary [1](#page-13-0) assume a single execution in which a single secret position (row, col) is shared across all gates and fresh permutations (π (i) in , π (i) out ) are sampled independently per gate. This is the correct description of the implementation.

Against an adaptive adversary who observes multiple executions, two requirements hold. First, the position (row, col) must be sampled freshly for each execution: position reuse across runs enables a differential observation attack in which Carol compares the same pixel across executions and identifies the consistent signal. Second, all permutations (π (i) in , π (i) out ) must be resampled independently for every gate in every execution: permutation reuse enables Carol to amplify the channel-matching signal of Theorem [2](#page-8-1) across runs. Fresh independent sampling of position per execution and permutations per gate per execution is therefore a strict implementation requirement for adaptive security.

## <span id="page-14-0"></span>6 Implementation

CRISP is implemented in Python 3.9+ using NumPy for vectorised array operations and PIL for image handling. The implementation consists of approximately 500 lines of code organised into five functional modules corresponding to the formal algorithms: key generation, embedding, computation, extraction, and testing. The complete source code is publicly available at [https:](https://anonymous.4open.science/r/CRISP-01F3) [//anonymous.4open.science/r/CRISP-01F3](https://anonymous.4open.science/r/CRISP-01F3).

#### 6.1 Key Design Decisions

The implementation differs from ProSt in three fundamental ways:

- Per-gate image assignment: ProSt maintains a dictionary mapping each wire to an image (61 images for the obfuscated circuit), while CRISP maintains a dictionary mapping each wire to a scalar value and processes one image per gate (17 images for the same circuit). This yields a 3.6× reduction in image count.
- Fresh permutations per gate: Unlike ProSt which assigns fixed permutations to wires, CRISP samples independent uniform-random permutations πin, πout ∈ S<sup>3</sup> for every gate. This provides full circuit privacy as the channelto-role mapping is fresh at both input and output of each gate.
- Secret position: The secret consists only of pixel coordinates (r, c), with no channel component. The channel dimension contributes no additional secrecy as permutations dynamically assign channel roles, giving n = h × w candidate positions.

#### 6.2 Core Primitives

The implementation provides four core functions that directly implement algorithms from Section [4:](#page-4-0)

{15}------------------------------------------------

- perm gen(): Samples two independent uniform random permutations from S3.
- emb(image, bits, pos, pi in): Embeds three bits into a single image at position (r, c) using πin for channel assignment.
- comp(stego, out cover, pi in, pi out): Vectorised Fredkin gate applied pixel-wise across the entire image, writing results to a fresh carrier.
- ext(image, pos, pi out): Extracts three output bits from position (r, c) using πout.

The computation is fully vectorised using NumPy operations, processing all pixels simultaneously without Python loops. For a 128×128 image, this achieves over 300× speedup compared to a naive pixel-wise implementation.

#### 6.3 Circuit Evaluation Pipeline

Carol evaluates the obfuscated circuit by:

- 1. Parsing the circuit specification (identical parser to ProSt)
- 2. Randomising gate execution order via randomised topological sort
- 3. For each gate in randomised order:
  - Retrieve input wire values from scalar dictionary
  - Sample fresh πin, πout using perm gen()
  - Apply emb → comp → ext using independent cover images
  - Update output wire values in scalar dictionary

All intermediate stego and output images are saved for verification, and optional visualisation is provided via matplotlib.

#### 6.4 Verification

The implementation includes exhaustive unit tests covering all 8 × 6 × 6 = 288 combinations of input bits and permutations, verifying that Ext ◦ Comp ◦ Emb correctly computes the Fredkin function for random secret positions. Full truth table verification confirms the circuit implements (A∧C)∨(¬A∧B)∨(¬B ∧¬C) correctly for all 8 input combinations.

## <span id="page-18-0"></span>8 Conclusion

We presented CRISP, a circuit-private single-image steganography scheme that extends ProSt with dual independent permutations. CRISP resolves two fundamental ProSt limitations: the O(L)-image overhead and the complete absence of circuit privacy.

The central technical contribution is the dual-permutation design. πin and πout are both public and required by Carol for correct evaluation. Circuit privacy arises not from hiding πin, but from resampling πout independently per gate: without this, the Fredkin gate's pass-through of the control bit enables a deterministic channel-matching attack that lets Carol identify the control channel by comparing input and output images pixel-wise. With πout resampled independently per gate, this correlation is broken (Theorem [2,](#page-8-1) Remark [1\)](#page-10-0).

Through a direct Bayesian calculation, CRISP achieves an adversary advantage of Θ(1/n) with n = h×w. The sole secret is the pixel position; knowing both permutations provides Carol with no advantage over random guessing of the position. Per gate, CRISP uses 2 images (1 input cover + 1 output cover) versus ProSt's 6 (3 input wire images + 3 output wire images), a 3× per-gate reduction. For the 17-gate benchmark circuit, the total image count drops from 61 to 34, a 1.8× circuit-level reduction, with consistent results across all tested circuit structures. All 288 exhaustive correctness cases pass, and Hamming weight invariance is confirmed across all experimental evaluations.

Limitations. The current implementation handles linear-chain circuits. General DAG support with proper wire tracking for fan-out and fan-in is left for immediate future work. Fresh independent sampling of all key components per execution must be enforced in practice; reuse creates an adaptive attack surface.
